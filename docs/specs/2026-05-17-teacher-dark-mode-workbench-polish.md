# Spec: 教师工作台夜间模式成熟化打磨

## Why

前几轮夜间模式优化已经从玻璃拟态、发光渐变转向石墨黑实体界面，但当前教师工作台仍存在边框噪音偏多、青绿色强调分散、右侧操作区权重偏高、加载/错误状态不够成熟的问题。继续修改前需要先冻结目标和红线，避免再次滑向“加效果”的方向。

## Goal Sentence

夜间模式要像一个成熟、低噪音、高效率的教师数据工作台：石墨黑实体底盘、少量青绿色信号、清晰状态层级、没有玻璃感和装饰性特效。

## Goals

- 降低视觉噪音：减少重复边框、过亮描边和卡片堆叠感。
- 收敛强调色：青绿色只用于当前选中、主状态、主操作和关键数据，不作为普遍装饰。
- 重新分配权重：中间“当前作业/课堂过程数据”成为第一主角，右侧操作区降噪。
- 补齐真实状态：加载、错误、空数据、复制邀请码反馈都要像成熟产品状态。
- 保持白天模式不被拖坏，移动端断点不出现文字挤压和操作丢失。

## Non-goals

- 不做玻璃拟态、blur、发光阴影、背景网格、装饰性渐变。
- 不重构业务数据、后端接口或数据库。
- 不把教师工作台改成营销页或大 hero 页面。
- 不引入新的 UI 框架、图标库或复杂动画。

## Scenarios

### Scenario: 正常读取中

GIVEN 教师打开工作台且已有作业  
WHEN 作业概览还在读取  
THEN 中间区域显示稳定的加载骨架/占位，不只是一行文字，右侧下一步显示“读取提交记录”。

### Scenario: 接口失败

GIVEN 教师数据或概览接口失败  
WHEN 页面展示错误  
THEN 错误文案使用业务语言，例如“课堂数据读取失败，请稍后重试”，可以保留技术状态作为次要信息，但不只显示 `请求失败 (500)`。

### Scenario: 选择作业

GIVEN 左侧有多个作业  
WHEN 教师选择一个作业  
THEN 当前作业卡片有明确但克制的选中态，中间区域优先展示作业标题、班级、任务数和过程数据。

### Scenario: 快捷操作

GIVEN 右侧有重置邀请码、复制邀请码、编辑题目、班级概览  
WHEN 教师浏览页面  
THEN 操作区不抢中间数据权重，按钮像工具命令而不是主内容。

### Scenario: 无数据

GIVEN 当前作业还没有提交记录  
WHEN 页面显示空状态  
THEN 空状态有清晰标题和说明，视觉上是系统状态而不是临时占位框。

## Impact

- Frontend:
  - `frontend/src/styles.css`: 夜间 token、按钮、状态标签、空/错/加载态、教师页三栏布局和响应式。
  - `frontend/src/features/teacher/TeacherPage.tsx`: 错误文案、加载骨架、空状态说明、状态 tone、操作反馈。
- Backend: 不改。
- Database: 不改。
- AI / standard data: 不改。
- Deployment: 只影响前端静态构建产物。

## Design Constraints

- 石墨黑底盘不超过 3 个主要深色层级：页面背景、侧栏/操作栏、中间内容面。
- 边框只能表达结构分区或可交互边界，不能每个子元素都强描边。
- 青绿色只保留给：
  - 当前导航/当前作业。
  - 关键成功或在线状态。
  - 主要操作按钮或关键数据点。
- 警告/错误/成功色只表达真实状态，不用于装饰。
- hover/active 以背景层级变化为主，避免发光和位移。
- 加载状态使用低对比 skeleton，占位尺寸稳定，不能造成布局跳动。

## Risks

- 当前后端偶发 500 会影响真实数据态复查。缓解：覆盖失败态、读取态、空态，并在能读到数据时复查有数据态。
- in-app browser 当前无法调整 viewport。缓解：做 CSS 断点检查，必要时另寻浏览器/Playwright 环境做移动端截图。
- `styles.css` 已较大，继续追加暗色覆盖会增加维护成本。缓解：优先整合已有暗色规则，减少重复覆盖。
