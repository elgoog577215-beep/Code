# 正向经验

## 2026-07-08 自动提交后自动推送

- 背景：项目规则原本写成“任务完成后自动提交但不推送，用户明确要求推送时再推送”，实际协作中用户多次在提交后继续补一句“推送”，增加了重复确认和流程断点。
- 触发：用户要求“往 a.md 里面加一下，就是自动提交之后就要自动推送。然后同时执行一下这一轮的推送”；实际文件对应项目规则 `AGENTS.md`。
- 过程：按规则先备份 `AGENTS.md` 到 `AGENTS.backup-20260708-140501.md`，再把 Git 规则调整为“自动提交成功后继续自动推送”，同时保留推送前 `git status --short --branch`、无关改动隔离、显式 stage、fetch 和避免强推等门禁。
- 结论：后续任务达到收束条件后，应自动提交并自动推送当前任务相关提交；推送不是省略检查，而是在检查通过后成为默认闭环的一部分。
- 复用：以后完成实现类任务时，先确认当前任务改动范围、提交并推送；若工作区有无关改动，只提交/推送当前任务相关文件，保留无关改动不碰。
- 位置：`AGENTS.md`、`AGENTS.backup-20260708-140501.md`。

## 2026-07-06 管理页系统阻断状态应收进侧栏

- 背景：班级名单页曾把系统 readiness / 阻断检查做成横向状态条，压在页面标题和名单维护工作区之间，视觉上像全局告警插入主任务流。
- 触发：用户指出“阻断”不应横着放在该位置，至少应放到侧栏等辅助区域，否则界面显得怪、主工作区被割裂。
- 过程：移除子管理页顶层横向 `ManagementStatusStrip`，把 readiness 变成班级/题库侧栏内的紧凑状态卡；同时把 `BLOCKED/FAIL/WARN` 等工程态翻译成教师可读文案，并补中英文词条。
- 结论：管理页的系统健康、AI 配置、阻断检查属于辅助诊断信息，不应横跨主内容上方；优先放入侧栏、折叠详情或管理首页入口，避免干扰当前表单/列表任务。
- 复用：以后做教师端管理页、资源页或配置页时，先判断状态信息是否是当前任务主线；若不是，放侧栏或折叠区，不再做横向页面切断条。
- 位置：`online-judge/frontend/src/features/teacher/TeacherManagementPage.tsx`、`online-judge/frontend/src/styles.css`、`online-judge/frontend/src/shared/i18n.tsx`。

## 2026-07-05 教师端直观性优先化繁为简和小方块展示

- 背景：教师端首页和作业详情曾把状态、下一步、作业列表、学生摘要多层堆叠，虽然信息完整，但重复内容多，横向长条过多。
- 触发：用户明确补充“直观”的定义：越少越好但保留必要功能和必要信息；适合改成方块的横向平铺内容应改成方块；能用小方块展示的不要用大方块。
- 过程：将教师首页收敛为班级/作业/学生/需看/新增的小状态方块、作业小卡片和需关注学生小卡片；作业详情保留优先学生和优先讲评题入口，同时把题目推进从表格长条改成题目小卡片。
- 结论：教师端直观性不等于增加醒目入口，而是先删重复、压信息，再用小卡片承载必要状态和动作；只有需要横向比较大量字段时才保留表格。
- 复用：以后改教师端或管理工作台时，先检查是否有重复信息层、是否有可卡片化的长条列表、是否存在过大的单信息块，再决定布局。
- 位置：`online-judge/frontend/src/features/teacher/TeacherPage.tsx`、`online-judge/frontend/src/features/teacher/AssignmentDetailPage.tsx`、`online-judge/frontend/src/styles.css`。

## 2026-07-05 标准库扩张采用模块化细颗粒条目

- 背景：用户明确要求可以直接修改 AI 标准库，让标准库继续扩张，并提升精度与颗粒度。
- 触发：在 V6 已经完成大规模模块化扩张后，需要继续补足同一主题内部更细、可诊断的常见错因，而不是继续堆泛化条目。
- 过程：新增 `AiStandardLibraryV7DetailSeeds`，沿用“1 个能力点 + 3 个易错点”的模块化结构，覆盖 DP、二分、前缀差分、图搜索、字符串、映射、数值边界和多组状态；每个条目都挂合法知识节点和能力点，并新增 V7 专项测试。
- 结论：标准库扩张优先补“高频、可观察、能区分相邻错因”的条目；不要写固定个性化建议，不要把条目命名成“理解或应用偏差”这类万能原因。
- 复用：下一轮扩库时先选主题，再按知识节点确认合法 code，最后补 seed、专项测试、总量门禁和 OpenSpec 验收。
- 位置：`online-judge/src/main/java/com/onlinejudge/learning/standardlibrary/application/AiStandardLibraryV7DetailSeeds.java`、`online-judge/src/test/java/com/onlinejudge/learning/standardlibrary/AiStandardLibraryV7DetailSeedsTest.java`。
