# 正向经验

## 2026-07-05 标准库扩张采用模块化细颗粒条目

- 背景：用户明确要求可以直接修改 AI 标准库，让标准库继续扩张，并提升精度与颗粒度。
- 触发：在 V6 已经完成大规模模块化扩张后，需要继续补足同一主题内部更细、可诊断的常见错因，而不是继续堆泛化条目。
- 过程：新增 `AiStandardLibraryV7DetailSeeds`，沿用“1 个能力点 + 3 个易错点”的模块化结构，覆盖 DP、二分、前缀差分、图搜索、字符串、映射、数值边界和多组状态；每个条目都挂合法知识节点和能力点，并新增 V7 专项测试。
- 结论：标准库扩张优先补“高频、可观察、能区分相邻错因”的条目；不要写固定个性化建议，不要把条目命名成“理解或应用偏差”这类万能原因。
- 复用：下一轮扩库时先选主题，再按知识节点确认合法 code，最后补 seed、专项测试、总量门禁和 OpenSpec 验收。
- 位置：`online-judge/src/main/java/com/onlinejudge/learning/standardlibrary/application/AiStandardLibraryV7DetailSeeds.java`、`online-judge/src/test/java/com/onlinejudge/learning/standardlibrary/AiStandardLibraryV7DetailSeedsTest.java`。
