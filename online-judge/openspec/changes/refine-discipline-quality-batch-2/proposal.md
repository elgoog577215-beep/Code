## Why

第一批学科精修已经清零结构性阻断项，但生产库仍有 562 个模板化知识点描述和 65 个缺少提升点的启用能力点。当前最需要继续提升的不是节点数量，而是数学、工程调试和程序基础高引用路径的教学语义，以及“知识点 → 能力点 → 易错点 → 提升点”的闭环完整度。

## What Changes

- 以生产库只读画像为基线，按正式标准库引用密度、领域覆盖缺口和教学中心性选择第二批知识点，不按领域平均凑数。
- 通过 Flyway V3 人工精修至少 30 个 MATH、ENG、BASIC 高价值模板知识点，补齐定义边界、可观察目标、典型任务、有效别名和真实前置知识。
- 为至少 12 个已有易错点但缺少提升点的正式能力点补充可执行提升路径，并同步规范化主表、兼容快照和兼容映射。
- 扩展学科质量门禁，验证第二批精修数量、规范化提升点闭环、兼容快照一致性和迁移前后核心业务计数。
- 在生产备份的隔离恢复库完成迁移演练，发布后通过 Flyway、质量门禁、内容 readiness、精确导航 API 和业务计数复核。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `discipline-data-quality-audit`: 增加分批质量债务下降、提升点闭环和规范化/兼容快照一致性的可重复门禁。
- `informatics-knowledge-tree-quality`: 要求后续人工精修批次按生产引用和领域缺口选择，并以批次版本记录可复核质量增量。
- `standard-library-normalized-schema`: 要求新增正式提升点同步规范化主表、兼容快照与兼容映射，并能从精确知识点导航读取。

## Impact

- 数据库：新增 Flyway V3，只更新正式知识树内容并新增提升点相关记录，不删除业务数据，不修改稳定 code。
- 脚本与测试：扩展 `scripts/check-discipline-data-quality.sh`、迁移链测试和数据库 readiness 断言。
- 生产环境：执行前生成可校验 PostgreSQL 备份并隔离恢复演练；应用 API 结构不变。
- 报告：更新学科质量报告，公开第二批增量、剩余债务和下一批优先级。
