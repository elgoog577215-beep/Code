# 设计说明

## 1. 扩展边界

V7 继续沿用 V6 的模块化扩展方式，不改历史条目、不迁移旧文件、不改变数据库表结构。新增内容放入独立类：

```text
AiStandardLibrarySeedCatalog
  -> highQualitySamples(...)
  -> compatibilityMistakePoints(...)
  -> AiStandardLibraryV6ExpansionSeeds.addTo(...)
  -> AiStandardLibraryV7DetailSeeds.addTo(...)
  -> generatedFullCoverage(...)
```

这样主 Catalog 仍然是唯一入口，教师端、种子器、质量报告和 AI 选择链路无需新增接入点。

## 2. 内容策略

V7 只补“同一大主题内部容易被 AI 混成一类，但教学上必须区分”的细颗粒度条目。每组采用：

```text
1 个能力点 + 3 个易错点
```

优先补强方向：

- DP：计数 DP、树形 DP、状态压缩 DP、答案位置与初始化。
- 二分：浮点二分、搜索边界、单调性来源、闭半开模板契约。
- 前缀差分：二维差分、前缀下标含义、差分还原顺序。
- 图与搜索：多源 BFS、0-1 BFS、最短路松弛、拓扑/连通状态。
- 字符串与映射：字符编码、子串边界、频次 map、键值含义。
- 数值与多组状态：乘法溢出、浮点比较、每组清空和全局状态残留。

条目必须表达可观察错误，而不是把“知识点理解不到位”写成万能原因。

## 3. 质量门禁

新增测试检查：

- V7 新增条目总量不少于 40。
- 每个 V7 易错点必须关联 V7 能力点和合法知识节点。
- 代表性细颗粒主题必须存在具体 ID。
- 条目名称不得出现“理解或应用偏差”等空泛模板。
- 总体手写高质量样本阈值继续提高。

## 4. 兼容性

V7 不修改 API、不改数据库、不改前端。所有新增条目只作为标准库候选进入现有链路；AI 仍根据题目、代码、判题结果和标准库候选动态生成诊断与建议。
