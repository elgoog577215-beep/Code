## Context

生产 V5 当前有 737 个知识树节点，其中 586 个启用叶子知识点；启用规范层有 140 个能力点、359 个易错点和 99 个提升点。前三批累计精修 84 个知识点，但还有 502 个叶子使用占位描述、41 个能力点没有提升路径。当前诊断事实数量不足以支持行为频率排序，因此第四批必须把“标准范围、结构缺口、人工学科判断”置于行为次数之前。

正式内容以 PostgreSQL 为主库，运行时 seed 已退出主链路；Flyway 迁移不可改写，生产发布必须从 V5 追加 V6。知识树稳定 code、规范化三层结构和旧平铺兼容接口均需保持。

## Goals / Non-Goals

**Goals:**

- 用高中信息技术与 NOI 两套标准建立可追溯的章节级覆盖矩阵。
- 精修 BASIC、MATH、ENG、CONTEST 各 12 个叶子知识点，使其能支持 AI 导航、学生解释和教师验收。
- 新增 22 个具体易错行为和 22 个可执行提升动作，优先关闭 BASIC、CONTEST 的正式提升路径缺口。
- 在生产 V5 快照上证明内容增长、质量债务下降、结构一致且业务事实不变。

**Non-Goals:**

- 不根据当前小样本推断学生错误率或教学效果。
- 不新增知识树节点、能力点或高中/竞赛平行分类。
- 不修改题库、提交、诊断事实、学生反馈和 AI 链路代码。
- 不把逐叶节点全部标成官方标准直接出处。

## Decisions

### 1. 标准映射细化到章节，不细化到每个叶子

V6 为教育部标准和 NOI 大纲补充 26 条章节级映射，使总映射从 13 条增至 39 条。章节是官方内容模块与本地树结构之间可核验的最细安全粒度；叶子继续通过祖先继承覆盖。备选方案是逐叶映射，但这会把本地教学拆分伪装成官方原文粒度，因此不采用。

### 2. 采用“四领域各 12 点”的内容矩阵

BASIC 覆盖输入、控制、数组、字符串、函数和数值范围；MATH 覆盖数论、计数、位掩码和坐标几何；ENG 覆盖错误分类、边界复现、状态追踪和复杂度；CONTEST 覆盖约束、输入输出合同、提交检查和失败复盘。候选必须已经存在、启用、仍为模板描述，并经人工逐条重写。平均数量只是验收下限，具体条目由学科连贯性和现有结构引用决定。

### 3. 扩充易错点和提升点，不复制已有能力点

现有能力层已经能承接本批主题，重复创建同义能力会降低导航精度。V6 因此向 22 个正式能力添加更细的易错行为和专项训练：14 个 BASIC/CONTEST 能力获得首条提升路径，8 个 MATH/ENG 能力获得与既有训练不同的第二路径。每个新增提升点只关联同一能力下的启用易错点。

### 4. 规范表、平铺快照和 legacy mapping 同步写入

新增易错点和提升点先写规范化主表，再用同 code 写入 `ai_standard_library_items`，最后建立同 code 的 `MAPPED` 映射。迁移内部校验与外部质量门禁同时检查归属、内容、引用和启用状态。

### 5. 生产快照是内容迁移的主验证基线

空数据库只能验证 Schema/Flyway 安全，无法验证依赖既有正式内容的 V6。实施先只读导出生产 V5，在隔离 PostgreSQL 恢复后执行 V6、重复执行原始 SQL、运行应用迁移和质量门禁；正式发布前再备份生产并记录业务表计数。

## Risks / Trade-offs

- [没有真实学习数据，优先级可能偏离未来高频错因] → 明确把本批定义为标准与结构驱动，不宣称学习效果；保留后续用真实诊断校准的入口。
- [第二提升路径与既有内容语义重复] → 每条新训练必须采用不同的练习对象、观察证据或验收动作，并由 code、名称和关联错因去重。
- [官方标准映射过度解释] → 只在章节级保存直接来源；竞赛实践类本地推导使用 `INFERRED`，不冒充官方逐项表述。
- [V6 在空库无内容可更新] → 空库用于迁移链安全，生产快照恢复库用于内容数量和导航闭环验证，两类结果分开报告。
- [生产迁移不可逆] → 发布前生成可校验备份；如 V6 失败，停止应用并恢复 V5 备份和上一镜像，不修改已发布迁移文件。

## Migration Plan

1. 固化生产 V5 只读快照、Flyway 历史、内容计数和核心业务表计数。
2. 新增并静态审查 V6，禁止删除、截断或修改业务表。
3. 在隔离恢复库执行 V6，运行质量门禁、代表性 SQL 导航和业务计数对比。
4. 运行 V1→V6 空库、重复启动、显式 baseline、Schema 漂移阻断和后端测试。
5. 提交并推送代码，生产发布前再次备份；部署应用触发 Flyway V6。
6. 核对 Flyway、质量指标、业务计数、应用/Nginx 健康与四领域代表性导航。

## Open Questions

无。真实学生数据到达后，应另开批次重新排序剩余 454 个模板知识点和 27 个提升路径缺口。
