## Context

当前后台链路已经具备搜索层和建议层，但两层职责仍然混在一起：搜索层提供候选后，建议层容易被标准库候选限制；建议层输出被拆成多个填空字段，学生看到的文字生硬；校验器对证据引用和标准库 ID 过于严格，导致有效诊断整包回退；库外错因只能停留在自然语言建议里，无法稳定吸收到本地标准库。

本次设计把链路拆清楚：搜索层负责找方向，建议层负责判断和表达，标准库成长 Agent 负责把半命中和未命中的发现转成可执行的本地标准库变更。

## Goals / Non-Goals

**Goals:**

- 让搜索层支持命中、半命中、未命中三态。
- 让建议层生成基础层、提高层、下一步行动三段自然语言学生报告。
- 允许 AI 否定标准库候选，避免标准库限制模型判断。
- 建立标准库成长 Agent，使 AI 发现的新错因能被吸收成候选或正式条目。
- 让成长 Agent 有写入本地标准库的能力，同时具备审计、门禁和回滚。
- 降低 evidenceRef、未知 ID 等小错误导致整包失败的概率。
- 建立真实样本回归集，持续验证正式链路质量。

**Non-Goals:**

- 不引入多校、多教师账号体系。
- 不做完全自动无审核的标准库大规模重写。
- 不在本阶段实现完整教程审批流。
- 不要求一次性解决所有模型供应商和价格策略。
- 不把标准库改造成规则引擎。

## Decisions

### 1. 搜索层只做导航，不做最终诊断

搜索层输出候选知识点、能力点、易错点、召回理由和三态判断建议。它可以提示“可能命中”，但最终是否命中由建议层确认。

替代方案是让搜索层直接决定错因。该方案更简单，但会把标准库变成隐性规则引擎，容易限制大模型判断，因此不采用。

### 2. 使用 HIT / PARTIAL / MISS 作为标准库关系协议

- `HIT`：标准库中存在足够准确的条目。
- `PARTIAL`：标准库方向正确，但颗粒度不够。
- `MISS`：当前标准库候选不合适，需要库外发现。

该协议使系统既能复用标准库 ID，又不会强迫 AI 选择不准确条目。

### 3. 建议层生成自然语言三段报告

学生可见内容只保留三段：基础层、提高层、下一步行动。内部仍保留 anchor、证据、置信度、命中状态等结构化信息，但不再把学生解释拆成大量固定字段。

提示安全边界调整为：允许知识方向、状态定义、小反例和手推方法；禁止完整代码、完整公式、完整答案和可照抄解法。

### 4. 标准库成长 Agent 采用“可写入但有门禁”的权限模型

成长 Agent 不只是生成建议文字，而是读取当前本地标准库，比较已有条目，生成具体变更草案。变更可进入两个目标：

- 候选池：默认目标，适合新错因、新能力点、新知识路径。
- 正式库：仅在满足严格门禁时写入，例如高置信、重复出现、无冲突、来源完整、教师确认或配置允许自动入库。

每次变更必须包含来源提交、来源题目、AI 理由、相似条目、差异说明、预期影响和回滚记录。

### 5. 标准库写入采用变更集，不直接散改

成长 Agent 生成 `StandardLibraryChangeSet`。后端先做预检，再决定保存为候选、等待教师确认，或写入正式库。

变更集至少包含：

- 新增、修改、合并或停用动作。
- 目标知识路径。
- 受影响条目。
- 与现有条目的相似度和冲突说明。
- 变更前快照。
- 可回滚操作。

这样可以让 AI 有足够权限执行变更，又避免本地标准库越来越乱。

### 6. 校验器改为硬失败与软修复两级

硬失败包括无法解析、无学生报告、明显答案泄露、完全无证据、与判题事实严重冲突。

软修复包括证据别名、未知标准库 ID、坏 anchor、提高层缺失。软修复不应导致整包回退，而应记录 trace，并尽量保留学生报告。

### 7. 真实样本作为质量门槛

采用 8 个样本作为第一批回归集：两数求和、二维差分、单调栈重复值、石子合并、动态连通性、字符串窗口边界、滑窗调平、分层最短路。

验收重点不是 JSON 是否好看，而是正式链路在准确性、自然表达、稳定性和可沉淀性上是否优于快反馈。

## Risks / Trade-offs

- 标准库成长 Agent 权限过大可能导致库污染 → 使用变更集、预检、教师确认、冲突检测和回滚快照。
- 允许自然语言报告可能降低结构化可控性 → 只把学生报告放宽，内部 anchor、证据、命中状态仍结构化。
- HIT / PARTIAL / MISS 会增加链路复杂度 → 用真实样本回归保证复杂度换来质量提升。
- 弹性校验可能放过部分低质量输出 → 保留硬失败门槛，并在 trace 中记录所有软修复。
- 模型 429 或超时仍会影响体验 → 本变更记录失败原因并保留后续模型路由接口，完整模型路由可单独推进。
