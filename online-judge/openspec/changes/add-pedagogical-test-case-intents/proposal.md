## Why

现有 45 个正式测试点只有输入、期望输出、顺序和隐藏标记，判题只能告诉 AI “第几个测试点失败”，不能说明该测试点覆盖的是样例理解、定义边界、结构反例、状态组合还是复杂度压力。上一轮已经补齐知识、能力、易错点、提升点和课堂/竞赛场景，本轮最有杠杆的增长点是让最上游的判题事实本身带有学科语义，同时严格保持隐藏数据不可见。

## What Changes

- 为正式测试点增加稳定的语义身份、评测意图、学习目标、竞赛角色、揭示策略和标准库路径映射，并精修现有 16 题、45 个测试点。
- 在判题结果中保存测试点身份和语义快照，使历史诊断仍可追溯当时使用的评测语义。
- 将安全的测试点语义送入 AI 自由诊断和最终诊断上下文；语义只描述“测试覆盖什么”，不得预判学生错因。
- 对隐藏测试点只提供泛化语义，不提供输入、输出、数值或可反推测试数据的细节。
- 增加独立质量门禁、Schema readiness、PostgreSQL 迁移验证和开发者回归夹具；没有真实学生数据时只验证结构与链路，不宣称教学效果。

## Capabilities

### New Capabilities

- `test-case-semantic-evidence`: 定义测试点语义模型、精细覆盖、判题快照、AI 消费和隐藏数据安全边界。

### Modified Capabilities

- `database-first-content-source`: 将正式测试点语义纳入数据库主库和可审计迁移边界。
- `diagnosis-evidence-quality`: 将测试点语义身份、快照一致性和证据引用纳入诊断证据质量门禁。
- `ai-prompt-context-quality`: 让自由诊断和最终诊断读取安全的测试意图，同时禁止把测试意图当作已证实错因或隐藏数据线索。

## Impact

- 数据库：新增 Flyway V8，扩展 `test_cases` 与 `submission_case_results`，回填 45 个正式测试点及历史判题结果。
- 后端：扩展测试点实体、判题结果实体、证据包、模型简报和提示词版本。
- 运维：新增测试点语义质量脚本，扩展 Schema readiness、迁移集成测试与部署前门禁。
- 兼容性：现有题目 API 和学生可见样例结构保持兼容；教师新建的未评审测试点允许暂时没有正式语义，但不能通过正式内容发布门禁。
