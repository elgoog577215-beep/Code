# 设计说明

## 1. 模块化方式

本轮不移动历史条目，避免引入大规模 diff 和回归风险。新增 V6 条目放入独立模块：

```text
AiStandardLibrarySeedCatalog
  -> highQualitySamples(...)
  -> compatibilityMistakePoints(...)
  -> AiStandardLibraryV6ExpansionSeeds.addTo(...)
  -> generatedFullCoverage(...)
```

这样主 Catalog 仍是唯一入口，数据库种子化、回退和旧测试都保持兼容；新增内容从 V6 开始有独立边界。

## 2. 内容扩张策略

V6 只补“高频、可诊断、能提升 AI 返回颗粒度”的条目，每组采用：

```text
1 个能力点 + 3 个易错点
```

优先分区：

- DP 经典：01 背包、完全背包、计数 DP、区间 DP、答案位置。
- 二分：位置二分、答案二分、check 单调性、返回边界。
- 前缀差分：一维差分、二维前缀、容斥、边界补零。
- 贪心：区间贪心、排序依据、并列规则、反例检查。
- 枚举剪枝：枚举对象、范围、去重、剪枝条件。
- 图论与数据结构：BFS、Dijkstra、MST、堆、栈队列、map/set。

## 3. 质量门禁

新增测试检查：

- V6 模块条目总数。
- 每个 V6 易错点都能关联能力点和知识节点。
- 每个重点主题至少有代表性条目。
- 新增条目不出现“理解或应用偏差”这类空泛名称。
- 总 seed 数和手写样本阈值继续提高。

## 4. 兼容性

V6 不改 API、不改数据库、不改前端。所有新增能力只作为标准库候选项进入现有链路。
