## Context

当前知识树已经用 `informatics_knowledge_nodes` 表达统一信息学科结构，并且节点具备 `aliases` 字段。AI 标准库已经按“知识树 → 能力点 → 易错点 / 提升点”同步到规范表，旧扁平表作为兼容入口保留。

用户本轮要求不是拆出“高中库”和“竞赛库”，而是把高中术语作为同一知识树的优先入口：高中有明确叫法时用高中主名，通用、竞赛、英文或教师习惯叫法作为副名可搜；高中没有覆盖的深层内容继续使用竞赛或通用术语挂在同一知识路径下。

## Goals / Non-Goals

**Goals:**

- 让高中技术选考常见术语能作为知识树主名或别名出现。
- 让高中术语、通用术语和竞赛术语命中同一组知识节点或标准库条目。
- 在既有知识树下补齐浙江高中真题高频但本地缺口明显的主题。
- 按主题包新增 V14 手写标准库条目，保持“能力点 + 易错点 + 提升点”的质量结构。

**Non-Goals:**

- 不新增高中库、竞赛库、画像表或运行时分流。
- 不修改 AI 诊断主协议，不把标准库变成硬规则引擎。
- 不新增前端页面或数据库表。
- 不把年份考频作为诊断证据；年份趋势只进入教学说明和补库优先级。

## Decisions

### 1. 统一知识树，主名优先高中术语

同一概念只保留一条知识路径。高中有稳定叫法时，节点主名优先使用高中术语，例如“链表”“区间调度与合并”；原来的“链式思想”“区间贪心”等进入别名。这样教师端默认展示更贴近高中语境，竞赛或通用叫法仍能通过别名和相关知识点命中。

替代方案是新增高中/竞赛 profile 或两棵树，但会让同一概念重复存在，后续 AI 召回、教师搜索和标准库成长候选都更容易分裂。

### 2. 高中缺口补在既有 topic 下

新增高中真题主题时优先挂入既有章节：排序类进入 `ALGO.SORT`，数据压缩进入算法思想或字符串/模拟相关路径，链表继续归入 `DS.LINEAR`，区间调度继续归入贪心。只有既有 topic 无法承载时，才新增同章节下的 topic，不新增新的上层领域。

### 3. V14 标准库采用主题包

延续 V13 的扩库方式，每个主题包至少包含 1 个能力点、若干具体易错点和 1 个提升点。能力点和易错点名称使用高中可读术语，描述中保留通用或竞赛副名，帮助搜索和 AI 理解。

### 4. 搜索复用现有字段

现有标准库搜索已经匹配 code、name、category、description、teacherExplanation、commonMisconception 和 knowledgeNodeCodes。V14 条目会把副名写入名称、描述、教师解释或知识节点 code；知识树节点则通过 aliases 保存副名。本次不改搜索 API。

## Risks / Trade-offs

- 高中主名可能比竞赛术语更窄 → 在 description、aliases 和相关知识节点中保留通用/竞赛叫法。
- 一次性补齐 22 个主题容易变成凑数 → 本轮优先补最明显缺口，并用测试约束条目质量。
- 知识树改名可能影响旧路径展示 → code 保持稳定，主名和 path 更新，旧叫法进入 aliases，降低兼容风险。
