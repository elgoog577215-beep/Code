# 优化 AI 提示词模板与 API 上下文

## 背景
当前 AI 主链路已经收敛为本地召回标准库候选、单诊断 Agent、学生端优先展示 `studentReport`。下一步不应继续增加 Agent，而应打磨正式发给外部模型的文字：提示词模板、题目/代码/判题上下文、标准库候选说明和学生可见文风。

## 目标
- 让模型明确：标准库服务于细颗粒定位和标准化命名，不是限制模型判断。
- 让学生可见反馈更像老师讲解：先讲人话，再给证据，再给下一步行动。
- 保持基础层优先，提高层作为修复后的学习提升。
- 允许 `OUT_OF_LIBRARY`，避免模型强行套用不匹配的标准库条目。
- 保持后端结构化输出与安全校验，不泄露完整答案或可复制改法。

## 非目标
- 不恢复双外部 Agent 默认链路。
- 不新增复杂 prompt 管理后台。
- 不改标准库结构。
- 不改外部模型 provider 选择逻辑。
