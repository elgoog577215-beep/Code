# 设计说明

## 1. 质量报告

新增一个标准库质量报告服务，直接基于当前知识树种子和标准库种子生成统计。第一阶段先用于后端测试和内部判断，不做前端页面。

报告结构包括：

- 总量统计：领域、章节、主题、细知识点、能力点、易错点、手写样本数量。
- 领域覆盖：每个领域的知识点数、能力点数、易错点数、手写能力点关联数、手写易错点关联数。
- 薄弱主题：按主题统计手写关联数，优先暴露手写覆盖为 0 或偏少的主题。
- 错因类型分布：统计易错点类型是否过度集中。
- 推荐项：把薄弱主题映射成下一批补库方向。

## 2. 手写高质量条目扩张

新增内容仍写入 `AiStandardLibrarySeedCatalog.highQualitySamples`，因为当前种子化和回退逻辑都依赖这个目录。

本轮优先补 P0：

- 模拟状态与流程。
- 字符串字符处理、匹配、构造。
- 数组遍历与原地更新。
- 读题建模与提交前检查。

条目遵守约束：

- 能力点是具体动作。
- 易错点是具体错误。
- 不写完整个性化答案。
- 通过知识节点和能力点形成可追踪关系。

## 3. 测试门禁

新增测试覆盖：

- 质量报告能统计出总体规模和领域覆盖。
- 报告能识别薄弱主题。
- 新增 P0 条目存在且语言具体。
- 手写高质量条目数量门槛提高。

## 4. 兼容性

不修改数据库表结构，不移除旧字段，不改变教师后台现有 API。新增服务只读取种子目录，后续可再升级为读取数据库与教师编辑内容。
