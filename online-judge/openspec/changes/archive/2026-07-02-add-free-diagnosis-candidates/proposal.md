# 增加自由诊断候选层

## 背景

当前单诊断 Agent 已经能接收本地召回后的树形标准库包，并输出 `studentReport`、`diagnosisDecision` 和 `libraryGrowth`。但真实评测暴露出一个结构性问题：模型可以看懂代码问题，却经常无法稳定绑定到标准库细颗粒 ID，或者为了满足结构字段而硬套标准库。

这说明标准库的角色需要进一步摆正：标准库应当是知识地图、命名词典和成长沉淀入口，而不是限制模型自由诊断的牢笼。

## 目标

在不恢复双外部 Agent、不增加实时调用次数的前提下，让单诊断 Agent 在一次输出中显式完成三步：

1. 自由列出它认为真实存在的诊断候选。
2. 再把这些候选映射到标准库：`HIT`、`PARTIAL`、`MISS` 或 `OUT_OF_LIBRARY`。
3. 最后写学生可见的自然反馈，并把库外发现送入候选池。

## 非目标

- 不新增第二个实时搜索 Agent。
- 不让 AI 实时自动修改正式标准库。
- 不改变学生端主展示结构。
- 不放松标准库 ID 和证据引用的后端校验。
- 不引入复杂评分器或新数据库表。

## 价值

- 保留模型自由分析代码的能力。
- 提升标准库命中、半命中、未命中的可解释性。
- 让库外发现更容易被吸收进标准库成长池。
- 避免模型为了填结构字段而牺牲学生反馈质量。
