## Context

现有标准库已经有多轮扩展，但扫描高考高频方向后，队列/循环队列、二叉树遍历、二叉树数组表示、栈表达式、滑动窗口计数、位置二分边界和递归分治仍有明显低密度点：有些知识点只有单个易错点，有些只有相关条目，没有面向高中题型的能力点和提升点闭环。

## Decisions

1. **直接补高价值薄弱点。**
   - 不先交付覆盖报告。
   - 扫描只作为内部优先级依据，输出直接是 seed 内容。

2. **继续沿统一诊断层建模。**
   - 每个新增主题都以知识点为主锚点。
   - 能力点下挂多个易错点和一个提升点。

3. **每个主题形成最小闭环。**
   - 1 个能力点回答“会做什么”。
   - 3 个易错点回答“常错在哪里”。
   - 1 个提升点回答“如何复盘迁移”。

4. **不新增数据库表。**
   - 这次目标是内容密度，不改治理表或教师端。
   - 通过现有 normalized seeder 同步进入正式表。

## Risks

- **条目重复**：V15 只补高频薄弱方向，名称和 code 使用 V15 前缀，避免覆盖旧条目。
- **条目泛化**：测试禁止模板化名称和描述，代表性断言检查具体症状。
- **锚点错误**：测试校验每条至少挂到一个 `KNOWLEDGE_POINT`。
