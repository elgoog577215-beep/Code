# 标准库主关系收束

## 背景
当前标准库同时存在知识树、能力点、易错点和兼容扁平条目。规范表已经有 `primaryKnowledgeNodeCode`，但兼容表、教师接口和运行时候选包仍主要暴露 `knowledgeNodeCodes`，导致产品理解上像“易错点既指向能力点，又指向知识树”的双重主关系。

## 目标
把主关系收束为：

```text
知识树节点 -> 能力点 -> 易错点
```

并保留辅助关系：

```text
相关知识点、前置知识、旧字段兼容
```

## 非目标
- 不重建所有标准库表。
- 不批量重写 1300+ 条种子内容。
- 不删除旧 `knowledgeNodeCodes` 字段，避免破坏现有数据、教师后台和兼容接口。
- 不调整 AI 诊断 prompt 的整体路线。

## 预期收益
- 教师和 AI 看到的标准库结构更清楚。
- 学生端知识路径能优先来自主知识点，而不是一堆平铺标签。
- 后续扩展时知道：能力点挂主知识点，易错点挂能力点，相关知识只做辅助。
