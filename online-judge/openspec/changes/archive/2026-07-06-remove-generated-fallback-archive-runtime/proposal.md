## Why

自动生成兜底层已经完成价值榨取和多轮手写吸收，现在继续把完整兜底库留在后端代码里，会让后续误接回运行链路的风险持续存在。用户希望在删除前本地备份，并允许通过 Git/云端同步备份目录，但备份永远不参与前后端。

## What Changes

- 在仓库内新增备份目录，保存删除前的自动兜底库快照和说明，作为纯备份材料。
- 从后端运行时代码中移除自动兜底存档枚举入口与价值分类入口。
- 保留自动兜底 code 识别能力，用于禁用历史数据库中的旧兜底记录，但不再提供任何兜底 seed 列表。
- 更新测试与规格，确认活跃标准库、候选包、质量报告和前后端入口不再依赖兜底存档。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `standard-library-normalized-schema`: 自动兜底从“代码内冷存档素材池”进一步降级为“备份目录文件”，运行时不得枚举、加载或分类兜底库。

## Impact

- 影响 AI 标准库 seed catalog、兜底存档分类类、标准库质量测试、OpenSpec 规格和备份目录。
- 不新增数据库迁移；历史数据库中已有兜底记录仍通过 code 识别并禁用。
- 备份目录可被 Git 同步，但不得被 Java 编译、Seeder、API、前端或 AI 候选包读取。
