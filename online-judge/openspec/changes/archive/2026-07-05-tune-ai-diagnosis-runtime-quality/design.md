# 设计说明

## Context

当前 AI 诊断主链路已经是单诊断 Agent：证据包和规则信号进入运行时，标准库作为教学参考规范包，最终由 `diagnosis-report-v2` 返回学生报告和后端结构信息。最新真实样题仿真显示链路可以跑通，但默认模型仍指向实测不可用的 DeepSeek Flash，且安全关键词对正常算法诊断存在误杀风险。

## Goals / Non-Goals

**Goals:**

- 默认模型使用已在真实链路中验证可用的 Qwen 主模型。
- 保留学生端答案泄露拦截，同时减少“合理诊断表达”被硬拦截。
- 让 `diagnosis-report-v2` 的 prompt 更清晰地区分学生自然报告和后端 metadata。
- 用单元测试固定这些边界。

**Non-Goals:**

- 不新增并行 AI 诊断模块。
- 不修改标准库数据库结构。
- 不改变“建议条数由证据决定”的原则。
- 不把标准库 ID 恢复为强制命中条件。

## Decisions

1. 默认模型只改配置，不在代码里硬编码模型路由。

   原因：环境变量仍然可以覆盖模型；本地、测试和部署都能继续按实际供应商调整。

2. 安全策略从“大词误杀”收缩到“直接答案泄露”。

   原因：算法诊断天然会出现“状态”“维度”“枚举顺序”等教学词，不能只因这些词出现就整包 fallback。直接代码替换、逐行改法、完整答案、可复制表达式仍然属于硬风险。

3. prompt 不删除结构化 metadata，而是降低学生文案复杂度。

   原因：后端仍需要 `diagnosisDecision`、`diagnosisCandidates`、`libraryGrowth` 做评测和标准库成长；真正需要简化的是学生可见表达和模型任务顺序。

4. 用测试锁定回归，而不是只靠真实模型仿真。

   原因：真实模型仿真成本高、受供应商波动影响；单元测试可以稳定覆盖默认模型、误杀边界和 prompt 关键措辞。

## Risks / Trade-offs

- [Risk] 安全校验变松后可能漏过直接改法。
  Mitigation：保留“完整代码、替换为、直接改成、逐行改法、代码块”等强泄露标记，并新增测试覆盖。

- [Risk] 默认模型未来在 ModelScope 下架或供应商策略变化。
  Mitigation：仍允许环境变量覆盖，真实评测报告继续记录模型可用性。

- [Risk] prompt 简化不足以改变模型行为。
  Mitigation：先用 prompt 单元测试锁定定位，后续继续用 8 题仿真看学生文案与 fallback 情况。
