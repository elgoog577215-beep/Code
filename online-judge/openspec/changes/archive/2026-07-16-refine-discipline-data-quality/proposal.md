## Why

生产库已经具备知识树与规范化标准库的基础规模，但真实画像显示，586 个细知识点仍使用同一套占位描述，697 个主题或知识点把父目录误写为前置知识，2,312 条启用关系指向已停用条目，且正式分类中混入大量 `V13/V14/兜底吸收` 等实现批次词。继续扩数量会放大学科语义噪声，因此当前应先把数据库从“有数据”提升到“学科边界明确、关系可信、可持续审计”。

## What Changes

- 以教育部高中信息技术课程标准与 CCF NOI 2025 大纲为双重参照，为统一信息学知识树建立可重叠的学科范围映射，不拆分高中库和竞赛库。
- 通过 Flyway 数据迁移清除伪前置关系、自指别名、失效关系和失效映射，并修复一条教师已批准但挂在占位路径上的正式易错点。
- 清理标准库分类中泄漏的内部版本和历史兜底批次词，保留稳定 code 以兼容历史引用。
- 首批人工精修高引用核心知识点的定义、学习目标、典型问题、别名与真实前置知识；其余模板债务保留为可量化待办，不批量生成低质量文本。
- 增加可重复运行的学科质量报告与硬门禁，区分阻断性完整性错误和渐进式内容质量债务。

## Capabilities

### New Capabilities

- `discipline-data-quality-audit`: 定义学科范围映射、数据库质量指标、严重度和发布阻断门槛。

### Modified Capabilities

- `informatics-knowledge-tree-quality`: 知识点必须区分父子归属与真实前置关系，并支持高中核心与竞赛层级的重叠映射及首批人工精修。
- `standard-library-normalized-schema`: 启用的能力点、易错点、提升点、关系和兼容映射必须只指向启用且存在的规范对象，正式分类不得泄漏内容批次实现词。
- `database-schema-migration`: 学科质量修正必须通过版本化 Flyway 迁移、隔离 PostgreSQL 验证和生产备份后发布。

## Impact

- 数据库：新增学科范围映射表并执行一次可审计的数据质量迁移。
- 运维：新增学科质量检查脚本，接入发布前验证；生产执行前必须备份并演练恢复。
- 内容：不改变知识节点和标准库稳定 code；会修正元数据、停用失效关系并重挂一条教师批准内容。
- 兼容性：现有导航与 AI 候选包继续读取同一规范化主表；无 API 破坏性变更。
