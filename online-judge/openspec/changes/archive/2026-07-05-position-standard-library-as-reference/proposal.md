# 将标准库定位为教学参考规范包

## 背景

当前 AI 诊断链路已经完成标准库规范结构收纳：知识树下面挂能力点，能力点下面挂易错点和提升点。这个结构的价值不是替外接大模型作答，而是给模型一份类似课程标准和教案目录的参考规范：帮助它统一术语、定位颗粒度、区分基础错误与提升方向，并为后续标准库成长留下可审核线索。

现有实现还有两类残留约束会削弱这个定位：

- 搜索定位层仍默认要求至少选择一个标准库候选，候选不合适时容易迫使模型硬套。
- 建议层逐条 advice 的标准库 ID 校验仍可能把未知 ID 当成整包失败，而不是把 ID 清空或转为库外发现。

## 变更目标

- 把 prompt 中的标准库语义统一为“教学参考规范包 / reference pack”，而不是答案库或强制命中表。
- 搜索定位层允许在候选不匹配、证据不足或需要扩库时返回空选择。
- 建议层保留对 JSON、安全、证据引用的硬校验，但把未知标准库 ID 从硬失败降级为软修复。
- 保持现有主链路，不新增并行 V3 模块，不拆出新的后端服务。

## 影响范围

- 后端 prompt 模板：`search-location-v1`、`diagnosis-report-v2`、运行时 user prompt 的 `contextPolicy`。
- 后端校验：`SearchLocationOutputValidator`、`AdviceGenerationOutputValidator`。
- 测试：prompt 注册断言、搜索定位输出校验、建议输出校验。

## 不做事项

- 不改变标准库数据库表结构。
- 不新增独立证据模式表。
- 不新增第三套 AI 诊断链路。
- 不把标准库成长候选自动合并为正式库条目。
