# 设计说明

## 核心定位

标准库是教学参考规范包，不是答案约束。它承担三件事：

- 给外接大模型提供知识树、能力点、易错点和提升点的参考地图。
- 规范模型输出的术语、颗粒度和基础层 / 提高层分工。
- 当真实问题没有被覆盖时，提供可审核的库外发现和后续扩库线索。

诊断权仍然属于正式诊断 Agent：模型必须先读题目、代码、判题结果和 evidenceRefs，再参考标准库映射；如果映射会牺牲真实判断，应使用 `OUT_OF_LIBRARY`、`MISS` 或空 ID。

## 搜索定位层

搜索定位层只负责整理参考包，不负责最终诊断。它可以返回 0 个候选，条件是输出明确表达：

- `libraryFit=MISS`，或
- `needsMoreEvidence=true`，或
- `needsLibraryGrowth=true`。

如果 `libraryFit=HIT` 或没有给出这些不匹配信号，则仍需要至少一个被证据支持的候选，避免模型空包误报成功。

## 建议层校验

硬校验继续保留在这些位置：

- 输出必须是可解析结构。
- 学生可见文本不得泄漏完整答案或逐行改法。
- evidenceRefs 必须来自当前 `ModelDiagnosisBrief`。
- confidence、必填字段和学生报告主结构必须合法。

标准库 ID 校验改为软约束：

- 未知 `mistakePointId`、`skillUnitId`、`improvementPointId` 不再导致整包失败。
- 校验器清空未知 ID，并在 `softFixes` 中记录。
- 诊断候选和诊断决策继续允许未知 anchor 转为 `OUT_OF_LIBRARY`。

## 兼容策略

旧 `diagnosis-and-advice-v1` 仍保留兼容，但它的未知标准库 ID 也要软清空，避免旧路径比正式 V2 更僵硬。旧扁平列表仍只作为合法 ID 参考，不改变主结构。

## 风险与验证

主要风险是校验变软后让低质量输出混入。控制方式是只放松标准库 ID，不放松证据、安全和结构校验。回归测试覆盖：

- 搜索定位 `MISS` 空选择通过，`HIT` 空选择失败。
- 建议层未知标准库 ID 被清空并记录软修复。
- prompt 明确标准库是参考规范包，并要求先自由诊断再映射。
