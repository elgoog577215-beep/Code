## Context

标准库当前存在两类内容：一类是 V6/V7/V8/V9 等人工精修智能条目，另一类是由知识树自动生成的兜底条目。兜底条目保证覆盖，但存在模板化表达、诊断动作不具体、教学提升不足的问题。它们更适合作为“知识点原材料”和“覆盖保障”，而不是长期作为高优先级诊断内容。

## Goals / Non-Goals

**Goals:**
- 建立兜底层吸收的第一期落地方式。
- 将一批高价值兜底知识点转成 V10 手写智能条目。
- 每个吸收主题包含能力点、多个易错点和提升点。
- 用测试保证被吸收知识点确实被 V10 手写条目覆盖。

**Non-Goals:**
- 不批量删除自动兜底条目。
- 不改候选包排序和 AI prompt。
- 不新增数据库字段来标记条目来源。
- 不一次性吸收全部 1000+ 自动兜底条目。

## Decisions

1. 先吸收，不删除。
   - 原因：自动兜底层仍然保证知识树全覆盖，直接删除会造成部分知识节点无标准库映射。
   - 取舍：本轮新增 V10 手写条目作为智能承接，旧兜底条目继续保留。

2. 使用 V10 独立 seed 文件承载吸收一期。
   - 原因：V6/V7/V8/V9 已经形成版本化扩库边界，V10 继续沿用，便于测试和后续归档。
   - 取舍：不把吸收条目混入基础手写样例区，避免主 seed catalog 膨胀。

3. 选择高价值模板化主题优先吸收。
   - 选择标准：自动兜底条目模板化明显、知识点高频、学生真实易错、能转化为清晰诊断和提升练习。
   - 本期覆盖：变量初始化/作用域、浮点精度、布尔条件、函数返回与副作用、数论取模、计数组合、错误类型定位、复杂度空间估算。

## Risks / Trade-offs

- [Risk] 新增智能条目与已有手写条目部分重叠 → 通过 V10 测试校验知识点承接关系，并让每个主题聚焦兜底层尚未精修的角度。
- [Risk] 自动兜底层继续存在会保留低质量文本 → 本轮先把高价值知识吸收到智能层，后续再做降权、合并或隐藏策略。
- [Risk] 条目继续增长导致维护成本增加 → 使用版本文件、专项测试和 OpenSpec 记录吸收范围。
