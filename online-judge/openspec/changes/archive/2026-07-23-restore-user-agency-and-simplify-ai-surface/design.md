## Context

当前产品已经具备稳定的判题事实、AI 诊断、标准库挂接、学生反馈、学习轨迹和教师分析能力，但后来新增的 Coach、复杂推荐编排、干预队列、自动标准库增长等模块形成了多条平行路线。学生首页把推荐结果作为唯一主要行动，题目结果把本次修正、成长信息和 Coach 放在同一层；教师端也出现了未接入正式路由的替代首页。问题不是 UI 元素数量本身，而是对象、证据和选择权的边界被打散。

## Goals / Non-Goals

**Goals:**

- 让用户先看到“我现在有哪些对象、各自是什么状态”，再自主选择进入哪一个。
- 让 AI 只解释事实、归因和可选方向，不把系统判断包装成唯一行动。
- 删除或下沉独立 Coach、复杂推荐编排、重复教师页面和自动标准库改写。
- 保留诊断、证据、历史数据以及必要的后台容错和质量工程能力。
- 任何页面都围绕一个明确对象和一种用户任务组织，不创建综合 AI 大面板。

**Non-Goals:**

- 不重做判题、核心诊断、标准库结构和教师对象层级。
- 不删除历史提交、历史 Coach 记录、推荐事件或成长候选数据。
- 不把教师分析简化为单个 AI 结论，也不新增聊天式 Agent。
- 不以“下一步”改名的方式保留原有强制推荐逻辑。

## Decisions

### 1. 学生首页以任务对象而不是 AI 推荐作为入口

首页默认展示课堂作业和自主练习两个用户可理解的任务集合，每项显示来源、完成状态和基础进度。系统不再请求学生推荐接口，不再生成一个占据首屏的“继续学习”推荐卡。若未来恢复个性化排序，必须提供排序依据和取消个性化能力，且不得隐藏完整任务列表。

选择该方案，是因为学生的真实任务是“选择作业或题目”，不是“消费一条推荐”。仅把推荐卡改名为“学习建议”仍然没有归还选择权。

### 2. 本次提交结果只回答本次提交

结果主界面保留判题事实、代码/测试证据、基础修复和提高建议。移除独立 Coach 生成、回答和换一问入口；成长趋势、错误生命周期和跨提交历史只通过独立“学习记录”入口访问。底部同时保留“继续修改”和“返回任务列表”，不得默认突出系统选择的下一题。

### 3. 推荐后台不再驱动学生主界面

停止学生首页对推荐 API 的调用和推荐曝光/点击事件生产。已有推荐、画像、误区、掌握度等内部服务暂不删除数据结构，但从产品主路径退出；后续在无真实使用证据前不再继续扩展策略。这样先切断错误的产品依赖，再安全清理后台代码。

### 4. 教师 AI 信息必须依附教学对象

正式路径保持班级 → 作业 → 题目 → 学生证据。删除未接入路由的 `TeacherPage`、`AssignmentDetailPage` 和旧 `ClassOverviewPage`，不把其中内容塞进新综合面板。需要关注的学生只能作为当前班级、作业或题目的可筛选证据状态出现。

### 5. 教师输出从必需模型阶段降为非阻塞能力

`TEACHER_OUTPUT` 不再参与诊断运行是否成功的判断。学生反馈完成只依赖核心诊断、标准库挂接和学生输出。教师分析继续从相同结构化事实确定性聚合；模型教师叙述可以暂时保留用于兼容，但失败不得拖垮学生链路。

### 6. 标准库增长只产候选，不自动改正式库

关闭默认自动合并，删除自动合并 HTTP 入口，并使候选生成流程只保存 `NEEDS_REVIEW` 等待人工批准。人工“批准/合并”入口保留并记录审计。相比提高自动门槛，完全取消自动晋升更符合标准库作为教学标准的责任边界。

## Risks / Trade-offs

- [学生失去个性化快捷入口] → 保留清晰的作业状态、最近活动和完整任务列表，让用户根据真实对象选择；待有真实行为数据后再评估可解释排序。
- [Coach 能力已有历史数据] → 只移除新入口和主链路调用，不删除历史表和记录。
- [旧推荐服务仍存在代码复杂度] → 本轮先停止产品调用和事件生产，后续以引用扫描和回归测试分批删除，避免一次性破坏轨迹聚合。
- [教师输出仍可能被内部代码引用] → 先改为非必需阶段并增加失败隔离，再根据真实调用清理生成阶段。
- [关闭自动晋升降低标准库增长速度] → 候选发现和聚合继续运行，人工审核台保留证据与批量治理能力。

## Migration Plan

1. 先调整前端入口和文案，使推荐、Coach 和重复页面退出用户主路径。
2. 再调整后端必需阶段、标准库自动合并配置与接口，保留兼容数据。
3. 更新测试，验证学生仍能自由进入任意作业/题目，AI反馈失败不影响判题事实，候选不会自动入正式库。
4. 构建前后端并运行 OpenSpec 校验。
5. 若需回滚，可恢复前端调用和路由；数据库无需回滚，因为本次不删除历史数据或表。

## Open Questions

无。本轮优先完成确定性的删减和边界恢复；是否重新引入可解释个性化排序必须等待真实使用数据。
