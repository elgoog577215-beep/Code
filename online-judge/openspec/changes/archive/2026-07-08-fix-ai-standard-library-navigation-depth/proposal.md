## Why

真实五题链路验证显示，AI 标准库导航稳定失败在 `STANDARD_LIBRARY_NAVIGATION` 阶段。根因不是标准库缺内容，也不是最终诊断 prompt，而是导航最大轮数与现有知识树深度不匹配：模型只能走到主题层，还没看到知识点诊断层就被迫结束。

这会让新的 `初步诊断 -> AI 标准库导航 -> 最终诊断` 主链路在真实模型下不可用，并诱发模型为了赶轮数选择未出现在当前视图里的 code，进一步触发 `INVALID_TAG`。

## What Changes

- 调整 AI 标准库导航循环，使它能覆盖根节点、大章节、小章节、知识点和诊断层的真实深度。
- 在每轮导航视图中显式提供当前可选知识节点 code 和诊断层 code，降低模型误选未展示 ID 的概率。
- 增加最后一轮与非法分支选择的护栏：模型不能在最后一轮继续返回 `CONTINUE`，非法分支选择必须被修正或失败关闭。
- 保持失败关闭原则：导航仍然不恢复本地召回，不生成缺少导航依据的学生可见诊断。
- 增加结构化回归测试，模拟 AI 沿知识树逐层导航到知识点诊断层并完成最终诊断。

## Capabilities

### New Capabilities

- 无

### Modified Capabilities

- `ai-diagnosis-orchestrator-v2`: AI 标准库导航必须支持真实知识树深度，并在每轮只允许模型选择当前视图明确暴露的标准库 code。

## Impact

- 影响后端 AI 诊断编排：`AiReportService` 的标准库导航循环、导航视图内容、非法选择处理和最终失败原因。
- 影响 prompt 约束：`standard-library-navigation-v1` 需要明确最后一轮、可选 code 和诊断层选择规则。
- 影响测试：新增/更新导航链路结构化测试，覆盖多轮导航、非法分支修正和失败关闭。
