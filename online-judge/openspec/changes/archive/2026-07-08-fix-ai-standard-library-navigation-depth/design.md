## Context

学生 AI 正式诊断已经切到 `初步诊断 -> AI 标准库导航 -> 最终诊断`。标准库结构是一棵知识树，路径通常为根节点、章节、主题、知识点，知识点下才有能力点、易错点和提升点。

当前导航循环固定 3 轮。真实树深下，3 轮通常只能走到主题层，模型还没有看到知识点诊断层就达到上限，最终以 `AI_NAVIGATION_ROUND_LIMIT_REACHED` 失败。真实模型还会因为轮数不足而尝试直接输出未出现在当前视图中的 code，触发 `INVALID_TAG`。

约束是：不能恢复本地召回，不能用后端规则替模型选择标准库路径；后端只负责展开、校验、修正非法结构和失败关闭。

## Goals / Non-Goals

**Goals:**

- 让 AI 导航能覆盖现有知识树真实深度，至少能从根节点走到知识点诊断层。
- 每轮把当前可选知识节点 code 和诊断层 code 明确传给模型，减少幻觉 ID。
- 对非法分支选择提供一次结构化修正机会；仍不合法则失败关闭。
- 保持外部模型唯一生产学生可见内容，导航失败不生成伪诊断。
- 用结构化测试模拟完整导航链路，覆盖根节点到诊断层的路径。

**Non-Goals:**

- 不恢复本地召回、关键词召回或后端打分。
- 不引入 AI 自查循环和候选自动入正式库。
- 不调整标准库数据库结构和 seed 内容。
- 不改变最终诊断报告的学生端展示格式。

## Decisions

1. 将导航最大轮数从硬编码 3 改为可配置默认 6。

   理由：现有树深至少需要根节点、章节、主题、知识点、诊断层五步；默认 6 给一次修正或较深路径留余量。相比无限循环，固定上限更容易控成本、控失败原因。

2. 每轮导航视图增加 `visibleKnowledgeNodeCodes`、`visibleDiagnosticCodes`、`mustFinishNow` 和 `navigationInstruction`。

   理由：模型最容易出错的是“知道某个概念，但该 code 当前没出现”。显式合法列表比只靠 DTO 嵌套字段更稳；`mustFinishNow` 能避免最后一轮继续返回 `CONTINUE`。

3. 后端在展开分支前先校验分支是否来自当前视图。

   理由：非法 code 如果直接进入 `expandKnowledgeNode`，异常会变成不清晰的服务失败。提前识别后可以让模型基于合法列表重选，或稳定返回 `INVALID_TAG`。

4. 非法分支只允许一次修正调用。

   理由：这能处理真实模型偶发幻觉，同时避免形成用户已经暂缓的自查循环。修正仍由 AI 选择，不引入本地召回。

5. 最终 `DONE` 仍通过标准库包和输出校验器确认。

   理由：导航循环只负责到达候选视图，不能放松标准库 ID、证据引用和置信度校验。若最终路径仍非法，链路按失败关闭处理。

## Risks / Trade-offs

- 轮数增加会增加一次到数次模型调用成本 -> 默认上限仍固定为 6，并只展开 AI 选中的一个主分支。
- 合法 code 列表会增加 prompt 体积 -> 列表只包含当前视图，远小于整库召回。
- 修正调用不能保证所有模型都改对 -> 修正失败仍失败关闭，不返回伪诊断。
- 如果未来知识树超过 6 层，仍可能到达上限 -> 配置项允许部署侧提高上限，后续可根据真实树最大深度动态计算。
