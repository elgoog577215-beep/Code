This file is a non-compilable text snapshot of removed legacy runtime code.

Archived sources:
- legacy long prompt branch inside enhanceSubmissionAnalysis(...)
- staged runtime: callDiagnosisJudgeStage -> callTeachingHintStage
- old single-call runtime: diagnosis-and-teaching-* -> CombinedOutput
- truncated diagnosisDecision extraction
- legacy AiAnalysisPayload parser

Key old entrypoints:

```java
private static final String RUNTIME_PROMPT_VERSION = "diagnosis-judge-v2+teaching-hint-v1";

@Value("${ai.external-runtime-mode:single-call}")
private String externalRuntimeMode = RUNTIME_MODE_SINGLE_CALL;

@Value("${ai.external-single-call-prompt-version:}")
private String externalSingleCallPromptVersion = "";
```

Legacy long prompt behavior:

```text
if shouldUseExternalRuntime(evidencePackage, ruleSignals) is false:
  build context from problem/submission/fallback
  call legacy long prompt
  parse AiAnalysisPayload
  build SubmissionAnalysisResponse directly
```

Staged runtime behavior:

```text
callDiagnosisJudgeStage(runtimePlan)
-> parse DiagnosisJudgeOutput
-> validateDiagnosisDecision
-> callTeachingHintStage(runtimePlan, decision)
-> parse TeachingHintOutput
-> validateTeachingHint
-> buildRuntimeAnalysisResponse
```

Old single-call behavior:

```text
callSingleCallRuntimeStage(runtimePlan)
-> parse CombinedOutput
-> validate diagnosisDecision / teachingHint / studentFeedback
-> buildRuntimeAnalysisResponse
```

Truncation recovery:

```text
if streamFinishReason == length:
  extract JSON object field "diagnosisDecision"
  parse DiagnosisJudgeOutput
  validate and use partial response
```

Reason for removal:
- These branches bypass or compete with the formal search-location + advice-generation chain.
- They produce old stage names and old prompt versions.
- They keep old DTOs alive in the runtime path.
