# 数据库学科质量阶段验收构建说明

## 报告任务

- 受众：产品与教学负责人。
- 决策：前三批学科内容升级是否站得住，当前应继续扩内容还是改变路线。
- 范围：比较生产 Flyway V4 证据基线与 Flyway V5 证据修复后的同口径状态；另行检查 V4 之后是否出现新提交和新分析。
- 成功标准：学科内容门禁继续通过；历史 advice 可确定性投影；临时路径有稳定身份；正式条目归属一致；反馈版本和事件关联完整；重复回填幂等；核心业务事实不变。
- 交付模式：自包含 HTML。当前运行时没有 Data Analytics MCP 报告渲染面，因此以规范 `artifact.json` 作为真源，并通过插件便携构建器生成。

## 结构映射

- 标题：`数据库学科质量阶段验收`。
- Executive Summary：标题后的首个正文区块，直接回答“成果如何、是否继续、为什么改变路线”。
- 关键发现与视觉证据：三种质量边界、证据资产前后对比、诊断路径构成、十二项阻断门禁、业务稳定性与演练纠错。
- 推荐动作：`下一步：先观察新样本，再决定第四批学科内容`。
- 进一步问题：`进一步问题`。
- 限制：`口径与限制`，同时在正式命中结论附近解释历史回填不能当新增效果。

## 口径约束

- `有 advice 分析投影完整率` 的分母只包含 21 份保存了 advice 的分析；12 份无 advice 分析不允许凭空生成事实。
- `稳定身份事实` 包含 `FORMAL_ID` 与 `PROVISIONAL_ID`，不包含 `TEXT_FINGERPRINT`。
- V5 新增的 26 条事实来自历史 advice 的确定性回填，不代表新增学生诊断。
- 正式路径 2→4 是历史投影补齐；由于 V4 后新提交和新分析均为 0，不能据此评价 V4 对正式命中率的影响。
- `30 个新分析` 是下一轮产品观察的务实触发线，不是统计显著性承诺；同时至少覆盖 5 道题和 3 名学生，才进入人工复核。
