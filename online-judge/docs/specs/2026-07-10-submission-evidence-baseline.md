# 学生提交证据闭环改造前基线

记录时间：2026-07-10

本文件保存 `build-student-submission-evidence-loop` 实施前的本地 H2 样本，用于解释新旧统计口径差异。数据来自改造前正在运行的本地服务，只作为迁移核对样本，不作为生产规模结论。

## 基线样本

| 范围 | 提交 | 具名学生 | 通过尝试 | 反馈记录 | 模型成功 | 真实查看 | 主要错因 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | --- |
| 作业 1 | 1 | 0 | 0 | 1 | 1 | 0 | `BOUNDARY_CONDITION` |
| 作业 2-4 | 0 | 0 | 0 | 0 | 0 | 0 | 无样本 |

作业 1 的教师概览曾把这条匿名课堂提交计入 `attemptCount=1`，但 `participantCount=0`、`submittedStudentCount=0`。这证明旧口径混合了“课堂尝试次数”和“可进入学生画像的合法提交”，也是本次改造必须拆分在册人数、合法提交人数与尝试次数的直接原因。

## 反馈与事件基线

- 作业 1：`submissionCount=1`、`failedSubmissionCount=1`。
- `feedbackRecordCount=1`、`modelReadyCount=1`、模型成功率 `100%`。
- `viewedCount=0`，因此不能把后台生成或预取解释为学生已查看。
- 唯一延迟样本约 `65,299 ms`，样本数过小，不能外推整体性能。

## 其他边界样本

- 公共练习：全库存量中有 94 次无作业提交，另有 1 次缺学生身份的课堂提交。公共练习不能和课堂作业历史混合；具名公共练习只按“学生 + 无作业 + 题目”恢复，匿名公共练习不归入任何学生。
- 失败反馈：全库当前反馈快照为 `READY=62`、`FAILED=33`。失败原因必须保留、重新打开不得自动重跑、显式重试成功后必须追加新版本而不是覆盖失败版本。
- 匿名公共练习：仍允许保留为匿名产品行为，但既不能恢复到某位学生的个人历史，也不能进入课堂学生画像。

上述全库分布在回填后通过只读 H2 查询复核；回填只补证据版本、诊断事实和事件关联，没有改写提交的作业归属、学生身份或反馈终态。不能为了补齐报表而伪造历史身份、失败率或公共练习归属。

## 回填后对照

| 证据对象 | 数量 | 完整性 |
| --- | ---: | --- |
| 提交 | 95 | 94 次公共练习；1 次课堂身份缺失 |
| 反馈版本 | 95 | 每个当前反馈快照均有可恢复版本 |
| 诊断事实 | 141 | 保留正式、临时、推断与未归类状态 |
| 反馈事件 | 110 | `feedback_revision_id` 缺失数为 0 |

## 迁移核对口径

1. 带 `assignment_id` 但缺 `student_profile_id` 的旧记录必须保留并标为身份缺失，不自动猜测学生。
2. 匿名课堂历史可以进入数据完整性分母，但不得进入涉及学生数、学生画像或学生通过率。
3. 反馈当前快照保持兼容读取；回填后应新增且只新增一个对应版本，重复回填不得增量。
4. 旧分析可以投影诊断事实；缺少可靠 breadcrumb 的事实必须标为 `INFERRED` 或 `UNCLASSIFIED`。
5. `AI_FEEDBACK_VIEWED` 仍以真实打开为准，后台预取不补写查看事件。

## 当前环境边界

本地数据库为 H2。实施前运行状态曾出现 H2 就绪提示，正式验证仍需同时通过实体启动、Maven 测试与 Postgres 兼容字段审查；本文件不把单次本地启动当成跨数据库验证。
