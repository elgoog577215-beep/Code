# AI 错题诊断链路说明

## 1. 当前结论

学生 AI 诊断只保留一条正式链路：

```text
学生提交
-> 判题事实整理
-> 初步诊断
-> AI 标准库导航
-> 最终诊断
-> 后端校验与留痕
-> 学生复盘报告
-> 标准库成长候选
```

这条链路的核心边界是：

- 初步诊断先看题目、代码和判题事实，不提前读取标准库条目。
- 标准库由 AI 按目录逐层导航，后端只展开 AI 选择的节点。
- 最终诊断同时读取原始事实、初步诊断和导航结果，生成学生可见报告。
- 后端负责结构校验、证据映射、标准库 ID 校验、失败关闭和成长候选记录。
- 老师负责审核成长候选是否进入正式标准库。

## 2. 为什么这样设计

标准库不是答案表，而是教学坐标系。AI 必须先理解当前提交的真实问题，再用标准库统一术语、颗粒度和成长线索。

如果后端在 AI 理解代码之前先替它缩小标准库范围，模型很容易围绕候选解释，而不是围绕学生的真实错误解释。现在的设计把“理解题目和代码”放在前面，把“标准库定位”变成 AI 的主动导航过程。

## 3. 阶段职责

### 3.1 判题事实整理

事实层只整理客观材料：

- 题目描述、输入输出、约束。
- 学生代码、语言、行号。
- 编译结果、运行结果、超时、错误输出。
- 失败样例、期望输出、学生输出。
- 可引用的证据片段。

事实层不输出教学诊断，不替 AI 判断错因。

### 3.2 初步诊断

初步诊断阶段只基于题目、代码和判题事实形成问题方向。

它可以输出：

- 可能的主因。
- 需要进一步确认的代码位置。
- 适合进入标准库导航的关键词或路径意图。
- 当前证据不足的地方。

它不输出学生可见复盘报告，也不写入正式标准库。

### 3.3 AI 标准库导航

AI 标准库导航阶段按统一知识树逐层读取：

```text
大章节
-> 小章节
-> 知识点
-> 能力点
-> 易错点 / 提升点
```

后端提供导航 API 和展开结果，不对候选做本地打分。AI 每轮只看到必要目录或节点摘要，避免一次性把整棵标准库塞进上下文。

导航结果应包含：

- 选中的知识路径。
- 选中的能力点、易错点或提升点。
- 标准库命中状态：`HIT`、`PARTIAL`、`MISS`、`OUT_OF_LIBRARY`。
- 如果标准库不够细，给出待审核成长候选线索。

### 3.4 最终诊断

最终诊断是唯一生成学生可见报告的阶段。

它读取：

- 原始题目、代码和判题事实。
- 初步诊断。
- AI 标准库导航结果。
- 标准库路径、能力点、易错点和提升点。

它输出：

- 基础层诊断。
- 提高层建议。
- 诊断证据。
- 下一步行动。
- 标准库命中状态。
- 成长候选线索。

最终诊断必须以当前提交证据为准。即使导航阶段选中过相近标准库节点，只要证据不支持，也不能强行命中。

### 3.5 后端校验与失败关闭

后端负责硬边界：

- 输出必须是合法结构。
- 学生可见内容必须引用真实证据。
- 标准库 ID 必须存在或被清空为库外线索。
- 不能输出可复制答案或泄题式改法。
- 初步诊断、导航或最终诊断失败时，不能伪装成 AI 成功。

## 4. 返回给前端的内容

后端最终返回的学生主内容应组织成自然文段，但保留结构化字段供前端和教师端使用。

示例：

```text
【基础层诊断】
你的主要问题在循环边界。代码第 18 行只处理到 n - 1，最后一个数据没有进入统计，所以在包含尾部最大值的测试点会少算一次。

【诊断证据】
题目要求统计全部 n 个数据；当前循环条件只覆盖前 n - 1 个位置。失败样例中最后一个元素正好影响答案。

【提高层建议】
这个问题对应“数组遍历边界控制”能力点。修完以后，可以再检查初始化、循环起点和终点是否和题目中的数据范围一致。

【下一步行动】
先用 3 个数手推一遍循环执行次数，写出每一轮 i 的值，再对照题目确认最后一个元素是否被处理。
```

结构化 trace 中保留：

- `standardLibraryNavigationStatus`
- `standardLibraryNavigationSelectedCount`
- `diagnosisLibraryFit`
- `evidenceRefs`
- `libraryGrowth`
- `fallbackUsed`

## 5. 标准库成长闭环

当 AI 判断现有标准库不够细或没有覆盖真实错因时，只能创建待审核候选。

成长候选应包含：

- 建议挂载的知识路径。
- 建议能力点或易错点名称。
- 典型错误表现。
- 学生解释话术。
- 来源提交证据。
- 相似已有条目。
- 审核状态。

正式标准库不能由实时诊断自动写入，必须经过教师审核。

## 6. 验收标准

- 生产代码只保留 AI 标准库导航相关阶段、配置和 trace。
- 默认学生诊断状态使用 `AI_NAVIGATION` 语义。
- 标准库导航失败时关闭或标记阶段失败，不回退生成学生可见诊断。
- 最终学生报告只来自最终诊断阶段。
- 标准库新增内容进入成长候选池，等待教师审核。
