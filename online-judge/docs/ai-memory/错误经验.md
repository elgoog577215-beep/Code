# 错误经验

## 2026-07-09 advice 安全重写不能只依赖模型自我纠错

- 背景：五道高级长代码 live 评测中，字段别名和建议数量已修复后，仍有两题在 `ADVICE_GENERATION` 第二次安全重写后失败。
- 错误：只靠 prompt 和二次请求要求模型“不要泄露答案”，模型仍会输出学生可见触发词，例如“将边的结束时间设为当前时间”“最终答案”，导致 validator 以 `SAFETY_RISK` 拒收整题。
- 结论：validator 不能放松；应在 `AdviceGenerationOutputNormalizer` 这个学生可见文案汇合点做受控清洗，去掉标题重复、直给修复语气和已知安全触发词，再交给 validator 严格校验。
- 复用：以后 live 报告出现 `studentReport/basicLayerAdvice contains answer leak`，先从 trace 精确定位触发字段和触发词，再把可恢复的表达清洗放到 normalizer；完整代码、参考答案等高风险内容仍应由 validator 拦截。

## 2026-07-09 全量测试依赖历史内容时必须显式开启 seed

- 背景：项目已决定生产默认关闭 `app.content-seed.enabled`，正式内容应来自数据库。但部分旧集成测试仍依赖知识树、标准库和教师端管理列表的历史 seed 内容。
- 错误：只跑局部 AI 链路测试会通过；一跑 `./mvnw -Dskip.frontend=true test`，依赖历史内容的测试会因为 H2 空库返回空树、空标准库列表或找不到相似条目而失败。
- 结论：测试如果验证“历史 seed 内容本身”或基于这些内容的管理接口，应在 `@TestPropertySource` 中显式设置 `app.content-seed.enabled=true`；测试如果验证新运行时主链路，则应自建最小 fixture，不能默认依赖启动 seed。
- 复用：以后看到知识树根节点为空、标准库列表为空、成长候选无法继承相似条目的 skillUnit，先检查测试是否应该显式开启 seed，而不是重新打开生产默认 seed。

## 2026-07-08 真实模型结构漂移不能只靠 prompt 约束

- 背景：高级长代码真实评测中，模型虽然理解了问题，但多次没有严格遵守完整 JSON schema：`outOfLibraryFindings` 返回字符串数组，`basicLayerAdvice` / `improvementLayerAdvice` 只返回 `text + evidenceRefs`，证据引用还会写成 `code:line_160`、`code:Line-160`、`judge:case_1_output` 等变体。
- 错误：只加强 prompt 或只让 validator 拒绝，会导致“信息足够但结构略偏”的真实输出被整体判为失败，最终学生端没有任何 advice。这会掩盖真正目标：保留可校验证据上的多条建议。
- 结论：对学生可见 AI 链路，应把 schema 分成硬字段和可恢复字段。证据不可校验、安全风险、空内容属于硬失败；`text-only advice`、字符串型库外发现、大小写/短横线证据别名、缺省 `confidence` 或 `nextStepPlan` 属于可恢复结构，应在 normalizer 中补齐或规范化。
- 复用：以后新增 AI 输出 schema 时，同步增加真实模型结构漂移测试；validator 负责拦截危险或无法确认的内容，normalizer 负责把可确认内容整理成内部稳定结构。

## 2026-07-08 外部模型测试必须先确认实际生效的 key 和模型

- 背景：五题 live 初次报告显示 ModelScope `401 Authentication failed`，但根因不是 `.env` 的 `MODELSCOPE_API_KEY` 失效，而是 shell 中已有的 `AI_EVAL_API_KEY` 优先级更高并覆盖了 `.env`。显式设置 `AI_EVAL_API_KEY="$MODELSCOPE_API_KEY"` 后，外部 API smoke 返回 200。
- 错误：只检查 `.env` 里 key 长度相同，不等于确认测试进程实际使用了那个 key。遇到 401/429 时，如果不先打印“变量名 + 长度 + 生效优先级”，容易把配置优先级问题误判成服务端不可用。
- 结论：真实模型评测前必须确认最终传入 Spring 测试的 `AI_EVAL_API_KEY`、`AI_EVAL_MODEL` 和 base URL；ModelScope 单模型 429 时，先用 `/models` 和小 token smoke 选可用模型，再跑长代码样本。
- 复用：后续跑真实 AI 评测，优先命令形态为 `AI_EVAL_API_KEY="$MODELSCOPE_API_KEY" AI_EVAL_MODEL='...' ./mvnw ...`，避免 shell 中旧 eval key 抢优先级。
