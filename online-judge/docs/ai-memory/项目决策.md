# 项目决策

## 2026-07-07 学生 AI 主链路移除本地规则信号

- 背景：`RuleSignalAnalyzer` 会把代码和判题结果压成 `candidateSignals`、粗标签和细标签；这在早期无外部大模型时有兜底价值，但在当前外部大模型主导诊断的链路里会造成先入为主，尤其容易让模型只围绕粗信号解释，忽略完整代码中的多重问题和库外问题。
- 决策：学生 AI 主链路不再把 `candidateSignals` 或 `RuleSignalAnalyzer` 输出传给外部模型，也不再用它们参与标准库召回加权。主代码已删除 `RuleSignalAnalyzer`、`RuleSignalResult`、`ruleSignals` 参数和 `ModelDiagnosisBrief.candidateSignals` 字段，历史实现只保留在 `legacy-archive`。模型直接读取题目、完整带行号代码、判题事实和树形标准库包；后端只负责结构校验、证据映射、标准库 ID 校验和成长候选记录。
- 复用：后续优化 AI 质量时，不要恢复本地规则信号作为模型输入。若需要提高命中率，应优先改标准库结构、树形召回、prompt 约束和评测闭环。

## 2026-07-07 学生快反馈代码证据由模型定位

- 背景：学生端 AI 反馈曾由后端先构造 `evidenceCandidates`、`sourceExcerpt`、`primaryRuntimeEvidence`，再要求模型绑定这些候选证据，容易把模型牵向后端猜测的少数代码行。
- 决策：学生快反馈实时链路直接把完整带行号源码 `submission.sourceCodeWithLineNumbers` 交给模型。模型负责阅读代码并返回 `code:line:N` 或 `code:range:A-B`；后端只做结构化校验、证据映射、可点击代码片段生成和安全兜底。
- 复用：后续优化 AI 诊断质量时，不要恢复 `E1/E2` 这类后端预选候选行。标准库和规则信号只能辅助模型理解方向，不能替模型决定错误行。
