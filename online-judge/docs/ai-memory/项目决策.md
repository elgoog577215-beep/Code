# 项目决策

## 2026-07-07 学生快反馈代码证据由模型定位

- 背景：学生端 AI 反馈曾由后端先构造 `evidenceCandidates`、`sourceExcerpt`、`primaryRuntimeEvidence`，再要求模型绑定这些候选证据，容易把模型牵向后端猜测的少数代码行。
- 决策：学生快反馈实时链路直接把完整带行号源码 `submission.sourceCodeWithLineNumbers` 交给模型。模型负责阅读代码并返回 `code:line:N` 或 `code:range:A-B`；后端只做结构化校验、证据映射、可点击代码片段生成和安全兜底。
- 复用：后续优化 AI 诊断质量时，不要恢复 `E1/E2` 这类后端预选候选行。标准库和规则信号只能辅助模型理解方向，不能替模型决定错误行。
