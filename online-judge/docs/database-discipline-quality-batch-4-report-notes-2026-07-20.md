# 第四批学科数据库扩充证据说明

## 结论

第四批是一次“标准驱动的高颗粒度扩库”，不是简单增加记录数。它在不增加知识树节点和能力点的前提下，把已有知识点写细、把章节标准映射补齐，并把能力下的“易错行为 -> 训练动作”补成可导航闭环。

## 选择依据

- 高中基础范围依据教育部《普通高中信息技术课程标准（2017 年版 2020 年修订）》中的数据与计算、程序控制、数据结构、调试和效率要求。
- 竞赛扩展范围依据 CCF《NOI 大纲（2025 年修订版）》中的 C++ 程序设计、数学基础和问题求解内容。
- 当前真实学生行为数据仍不足以代表总体错误率，因此只用生产结构引用定位已有承接点，不把小样本频次解释成教学效果。
- 官方出处只下沉到章节级；`CONTEST.READING`、`CONTEST.SUBMIT` 这类本地教学拆分标记为 `INFERRED`。

| 指标 | V5 基线 | 隔离库 V6 | 变化 |
|---|---:|---:|---:|
| 章节级标准映射 | 13 | 39 | +26 |
| 累计人工精修知识点 | 84 | 132 | +48 |
| 模板化叶子描述 | 502 | 454 | -48 |
| 启用易错点 | 359 | 381 | +22 |
| 启用提升点 | 99 | 121 | +22 |
| 缺少提升路径的能力点 | 41 | 27 | -14 |
| 知识树节点 | 737 | 737 | 0 |

48 个知识点按 `BASIC / MATH / ENG / CONTEST` 各 12 个精修；22 个易错点全部落到可观察的具体失败行为，22 个提升点全部包含练习方法和验收动作。14 个 BASIC、CONTEST 正式能力首次获得提升路径，另有 8 个 MATH、ENG 能力获得第二种专项训练。

## 代表性闭环

- BASIC：`二维数组边界检查 -> 方向移动后先访问再判界 -> 固定“生成坐标、判界、访问”顺序`。
- MATH：`子掩码枚举 -> sub=0 后回绕或死循环 -> 手算完整子掩码与终止轨迹`。
- ENG：`最大输入调试 -> 只估平均规模 -> 建立最大规模时间与空间总账`。
- CONTEST：`隐藏特殊条件 -> 把样例偶然规律当题目保证 -> 分离题面保证与样例观察证据`。

## 验证边界

生产 V5 快照已在本机隔离 PostgreSQL 恢复；V6 原始 SQL 连续执行两次后数量稳定，结构质量门禁全部通过。生产发布后 Flyway 为 V6，四个代表分支均通过真实教师导航接口读取；题目 16、测试用例 45、提交 53、分析 34、诊断事实 85、学生反馈 53，发布前后完全一致。应用端口、Nginx 和公开 HTTP 页面均返回 200。

本批只能证明数据库内容和工程质量提升，不能声称学生已经学得更好。后续应在真实新 cohort 足够后，用正式命中、重复临时候选和教师校正重新排序剩余 454 个模板知识点与 27 个路径缺口。
