# 新增外部模型路由池

## 背景

10 条 20 行以上长代码真实 live eval 已经满足覆盖底线，但报告显示外部完成率只有 20%：前 2 条由 ModelScope DeepSeek-V4-Pro 完成且质量、安全、证据和教学动作均达标，随后主路由返回 `insufficient_quota`，后续样本被预算门禁阻断。

这说明当前主要瓶颈不是提示词或标准库，而是线上外部模型路由容量。已有实现支持一个 fallback 路由，但学校部署时可能需要多家供应商、多 key 或不同模型级别共同兜底。

## 目标

- 在保留现有主路由和 `ai.fallback.*` 的基础上，新增可选多路由池。
- 提交诊断、Coach 追问和成长报告共用同一套路由枚举逻辑。
- 每条路由继续独立使用预算门禁，某个 provider/model 被 quota 或限流阻断时，不影响后续可用路由。
- live eval 报告继续按实际 provider/model/route role 归因，避免把容量失败误判为本地规则质量。

## 非目标

- 不做多模型投票、结果融合或自动择优。
- 不改变诊断协议、提示词标准库或安全校验。
- 不要求现有部署立刻改配置；未配置路由池时行为与当前一致。

## 影响范围

- 外部模型路由配置和解析。
- `AiReportService` 与 `CoachAgentService` 的路由枚举。
- 外部模型路由 fallback 相关单元测试。
- live eval 路由画像与容量瓶颈判断。
