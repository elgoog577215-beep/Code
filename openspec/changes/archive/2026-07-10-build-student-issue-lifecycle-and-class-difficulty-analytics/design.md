## Context

现有 `SubmissionDiagnosisFact` 已按提交保存 `REPAIR`、`IMPROVEMENT` 或兼容诊断事实，教师聚合已经返回错误出现次数、涉及学生数、重复学生数、涉及题目数和相邻同题恢复状态。但 `factKey` 包含分析 ID 和序号，只保证单次投影幂等，不能作为跨提交问题身份；学生反馈 DTO 也没有持续、新增、复发等生命周期信息。当前班级聚合中的出现次数还没有区分完全相同代码的重复提交与发生有效修改的独立尝试。

## Goals / Non-Goals

**Goals:**

- 每次提交完整保留全部诊断点，并为每个点生成可解释的跨提交规范化键。
- 按学生、作业、题目和提交顺序投影问题转换，支持新出现、持续、未观察、恢复、复发和不可比较。
- 从原始事实与转换记录派生个人持续难点、复发易错点和跨题薄弱点。
- 为班级同时提供去重覆盖、原始出现、有效加权出现、未解决、复发和恢复指标。
- 保持当前反馈快照、历史读取和教师三级下钻兼容。

**Non-Goals:**

- 不使用大模型额外调用来判断问题是否相同。
- 不把一次新问题直接标记为易错点或长期能力缺陷。
- 不把提升点与导致错误判题的问题混为同一错误分数。
- 不在本次变更中扩展多学科、生成教师教学决策或建立不可解释的综合排名。

## Decisions

### 1. 原始事实与生命周期投影分离

`SubmissionDiagnosisFact` 继续作为不可覆盖的逐提交事实，并新增 `normalizedPointKey`、`displayCategory` 和投影版本。新建 `SubmissionIssueTransition` 保存相邻提交间的转换，而不是回写旧事实状态。个人难点和班级重难点从事实与转换重算，规则调整时不迁移历史原始数据。

替代方案是直接在事实表维护当前状态，但这会覆盖历史语义，并且无法同时表达“当时是什么”与“后来如何变化”，因此不采用。

### 2. 规范化键优先使用正式身份

规范化键按以下优先级生成：`mistakePointId`、`improvementPointId`、`skillUnitId + factType`、临时/推断知识路径末级与规范化标题指纹。通用 `I1`、`I2` 等单次 issue 序号不得独立作为跨提交身份。键生成算法带版本号，并保留来源状态。

标准库与临时路径均缺失时，只能生成低置信度文本指纹；这类匹配显示为推断，不能伪装成正式同一问题。

### 3. 转换以同一学生同一作业同一题目的相邻可比较提交为主

每次分析事实投影成功后，生命周期服务按提交时间重算当前学生与题目范围。当前集合与上一可比较集合的交集为 `PERSISTED`，新增集合为 `NEW` 或历史曾出现时的 `RECURRED`，消失集合为 `NOT_OBSERVED`；当后续评测通过且诊断完整时，错误点可升级为 `RECOVERED`。诊断缺失或失败时记录 `UNCOMPARABLE`，不得推断恢复。

跨题重复从同一学生的规范化键涉及题目集合派生，不强行建立跨题相邻转换。

### 4. 完全相同代码保留原始行为但不放大有效权重

生命周期投影保存源码规范化哈希和 `effectiveAttempt`。完全相同源码、相同题目和相同学生的重复提交仍计入原始提交和原始出现次数，但有效加权出现次数只计第一次；源码、评测阶段或诊断集合发生变化时视为有效尝试。

这同时满足审计真实性与防止重复点击放大班级难度。

### 5. 事实状态与教学标签分层

转换状态是客观事实。个人标签按可配置阈值派生：一次出现为单次观察；连续有效出现达到默认 3 次为持续难点；消失后再次出现为复发易错点；涉及至少 2 道题为跨题薄弱点；有后续评测与诊断证据时为已恢复。页面同时展示次数和证据，不只展示标签。

班级不先合成黑盒分数，而是并列返回去重涉及学生数、原始出现次数、有效加权次数、重复学生数、当前未解决学生数、复发学生数和恢复率，并派生四象限分类：个别偶发、普遍易错、个体顽固、班级重难点。

### 6. 学生完整展示，变化信息作为附加层

当前反馈中的全部修复项和提升项继续完整显示，不按数量截断。生命周期服务在读取时为每个当前项补充规范化键、变化状态和次数，并返回上次存在但本次未观察或已有恢复证据的完整列表。前端通过分组和状态徽标降低认知负担，但不得隐藏任何当前项。

### 7. 幂等回填与兼容读取

历史回填先补事实规范化键，再按学生与题目重建转换。转换使用确定性唯一键，重复执行不得增量。读取开关关闭时继续返回原反馈和现有教师统计；新增字段为空不破坏旧前端。

## Risks / Trade-offs

- [文本指纹可能误合并或漏合并] → 优先正式 ID，低置信度匹配明确标为推断，并允许教师校正后重算。
- [同一问题在不同分析版本中拆分粒度不同] → 保存匹配算法和诊断版本，无法一一对应时使用 `UNCOMPARABLE`，不强行合并。
- [多次重算带来查询成本] → 只在单个学生/题目范围重算，增加规范化键、学生/题目/提交索引，班级聚合批量读取。
- [固定阈值把短期现象标签化] → 阈值配置化，前端优先展示“连续出现 3 次”等事实，标签作为辅助。
- [相同代码重复提交仍可能有不同评测结果] → 原始次数始终保留；评测阶段变化时仍可视为有效观察。

## Migration Plan

1. 以可空兼容字段和新转换表上线结构，现有读取保持不变。
2. 新提交在事实投影后同步补规范化键并异步/事务后重算生命周期。
3. 运行 dry-run 统计缺键、低置信度匹配、不可比较和重复代码范围。
4. 分批幂等回填历史事实与转换，核对重复执行计数不增加。
5. 开启学生生命周期读取，再开启教师双口径读取；异常时关闭新增读取开关，不删除证据。

## Open Questions

- 默认持续难点阈值采用 3 次有效提交、跨题阈值采用 2 道题，后续应由真实课堂数据校准。
- 提升点的“已采用”当前只使用代码与后续诊断证据，不等同于教师确认掌握。
