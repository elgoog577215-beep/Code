## Context

登录学生首页目前先展示大型“AI 学习导航”卡，再展示一个混合了公共题库和课堂作业的“学习任务”列表。推荐卡默认展开主标题、原因、标签、判断依据、完成标准、风险、回退动作和次级推荐；移动端几乎整个首屏都被该卡占据。与此同时，公共题库作为置顶大行获得了与课堂作业相同甚至更高的视觉权重。

推荐数据和事件追踪已经有价值，本变更调整的只是首页信息归属和渐进披露，不删除字段、不改变算法。

## Goals / Non-Goals

**Goals:**

- 学生五秒内知道当前主行动、课堂作业状态和自主练习入口。
- 桌面端形成三个清晰区块，移动端第一屏同时露出主行动和课堂作业标题或首行。
- 推荐主行动高度显著降低，仅保留标题、原因、轻量目标和按钮。
- 课堂作业优先于公共题库，公共题库成为轻入口。
- 推荐异常安静降级，不阻塞作业与自主练习。

**Non-Goals:**

- 不删除推荐字段、次级推荐或推荐事件。
- 不增加新的“推荐详情”页面；隐藏信息仍由题目结果、成长视图和后台证据消费。
- 不修改未登录首页、公共题库详情、作业详情、题目页和反馈弹窗。

## Decisions

### 1. 首页按任务所有权分区

登录首页固定为“继续学习、课堂作业、自主练习”。不再把公共题库和课堂作业混入同一列表，因为两者的来源、优先级和用户意图不同。

### 2. 推荐能力保留，解释层不在首页展开

主行动使用后端第一条推荐，继续保留 recommendation token。默认只显示标题、原因、完成信号和操作；`focusTags`、`learningHypothesis`、`riskLevel`、`fallbackAction` 与次级推荐不渲染在首页，但不从类型和接口移除。

### 3. 状态降级以用户任务为中心

加载时使用与最终卡片等高的轻骨架；无可靠建议时显示“继续课堂作业”并定位到作业区；推荐失败时不展示技术错误卡，而显示相同的任务入口和一条低权重说明。

### 4. 公共题库变成单行自主练习入口

登录首页不展示难度分布和置顶标记，只显示“公共题库”、题目数量和入口。未登录页面继续使用现有完整公共练习模块。

### 5. 视觉验证以首屏占用为门禁

桌面和 390px 移动端均截图验证。移动端的继续学习区不得占满首屏，课堂作业标题或首条作业必须在初始视口中出现。

## Risks / Trade-offs

- [隐藏解释后学生不知道推荐来源] → 保留一句具体原因和完成目标；完整证据仍在进入题目后的反馈页面。
- [暂无推荐时顶部区域价值降低] → 退化为继续课堂作业入口，而不是保留空卡或错误卡。
- [公共题库曝光下降] → 保留独立自主练习区，但明确低于教师作业优先级。
- [旧浏览器测试依赖混合任务列表] → 更新测试为三个功能区契约，并保留公共题库可访问性断言。

## Migration Plan

1. 先更新学生首页浏览器契约，锁定三个功能区及推荐披露边界。
2. 重组 `StudentPage` 登录态结构与双语文案。
3. 以首页作用域 CSS 重做桌面和移动端布局。
4. 完成测试、类型检查、构建与新旧截图对比。
5. 归档 OpenSpec，提交并推送主分支。
