## Why

当前 live-model core set 已能验证外接模型是否可调用，但样本数量偏少、代码形态偏干净，不能充分检验 40-80 行、多函数、多错因学生提交中的根因选择能力。人工目前无法介入，因此需要一套可复现的自动生成、自动执行、自动淘汰低质样本机制。

## What Changes

- 新增自动生成的复杂学生提交评测集，首批固定产出 100 条复杂静态诊断样本。
- 从 100 条样本中标记 24 条 `complex-live` 候选，并支持先跑 6 条真实 ModelScope live eval。
- 新增生成器与质量门禁：题目种子、正确解法、bug 注入配方、本地 Python 执行验证、重复与低质量过滤。
- 新增 loader 和结构测试，确保每条样本包含主错因、次错因、干扰信号、教学优先级、证据引用和安全禁止项。
- 不引入未知来源学生代码，不把外接模型输出作为标签真值。

## Capabilities

### New Capabilities

- `autogenerated-complex-eval-fixtures`: 自动生成并验证复杂学生提交诊断样本。

### Modified Capabilities

- `live-model-core-eval-fixtures`: live-model eval 可以纳入自动生成的复杂候选样本，并通过 case id 过滤小批量运行。

## Impact

- 新增测试资源：`diagnosis-eval-fixtures/complex-student-submission-cases.json`
- 新增测试生成器：`src/test/resources/diagnosis-eval-fixtures/generate_complex_student_submission_cases.py`
- 新增测试 loader 与断言：复杂 fixture loader、生成质量测试、live eval 接入测试
- 验证：相关后端测试、OpenSpec strict validate、secret scan、`git diff --check`，可行时跑 6 条真实 ModelScope smoke
