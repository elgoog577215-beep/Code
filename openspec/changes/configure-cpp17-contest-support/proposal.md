## Why

信息竞赛场景下，C++17 是学生训练和比赛最常用的主语言；平台如果只把 C++ 当成隐藏的执行器配置，会在课堂使用时暴露为模板、编译参数、环境检测和错误提示不一致。

当前项目已经有 Python 与 C++ 的基础执行能力，但 C++17 还没有被沉淀成一等竞赛语言能力。本变更要把 C++17 配进主链路，让学生提交、教师检查、后端判题和沙箱执行都围绕同一套语言契约工作。

## What Changes

- 将 C++17 定义为信息竞赛主语言之一，与 Python 3 一起进入正式提交白名单。
- 学生端默认语言调整为 C++17，贴合信息竞赛训练入口；Python 3 继续保留为可选语言。
- 统一本地执行器与 Docker 沙箱的 C++17 编译参数：`-std=c++17 -O2 -pipe`，确保课堂本机开发和部署沙箱行为接近。
- 抽出后端语言配置入口，减少 `54`、`C++17`、文件扩展名、编译器可用性文案散落在服务中。
- 强化执行环境状态：教师端和提交链路都能明确知道 C++17 是否可用，并输出面向课堂的错误说明。
- 强化学生端代码入口：C++17 下拉项、默认模板、文件名、编辑器语法和恢复模板保持一致。
- 增加 C++17 判题覆盖，验证 AC、编译错误和环境不可用等核心路径。

## Capabilities

### New Capabilities

- `contest-cpp17-execution`: 信息竞赛 C++17 提交、编译、执行、环境检测和学生端入口的一等支持。

### Modified Capabilities

无。本变更新增语言执行契约，不修改既有 OpenSpec 正式能力目录。

## Impact

- 后端：影响 `execution`、`submission`、`system` 相关语言配置、执行器和判题服务。
- 前端：影响学生题目页的默认语言、语言选择、模板恢复、编辑器模式，以及教师端执行环境展示文案。
- 测试：新增或更新 C++17 相关后端单元测试；前端变更需运行 typecheck。
- 部署：正式课堂建议使用 Docker 沙箱；本机模式仍可用于开发，但必须检测 `g++` 可用性。
- 非目标：本轮不开放 C 语言，不新增数据库表，不改变 AI 诊断结构，只确保 C++17 判题事实稳定进入后续 AI 链路。
