## Why

当前学生端仍以邀请码为主入口，公共题库、课堂作业、学生身份和教师工作台混在同一组导航语义里。学校真实使用时，学生和老师本来就会访问不同网址，不需要先经过一个角色选择首页。

本轮要把产品链路从“邀请码驱动”调整为“固定学生网址 + 固定教师网址 + 学生班级身份 + 作业/题库二分”，让同学进入后能自然找到下一步，同时保留现有判题和 AI 诊断能力。

## What Changes

- 学生端与教师端以固定网址明确分离：学生端主地址为 `/app/student`，教师端主地址为 `/app/teacher`；不再保留真实使用中的角色选择首页。
- 学生端新增“班级 + 姓名 + 学号/座号”登录链路，由教师端维护的班级名单承载身份来源。
- 未登录学生可以浏览公共题库；登录后可以同时看到公共题库和所在班级的老师作业。
- 老师作业列表只展示学生所属班级下的可见作业；一份作业包含多道题，作业详情页负责展示题目列表、完成状态和下一题。
- 公共题库作为“公开练习集”处理，不强制绑定作业；进入题目页时仍使用现有提交接口。
- 学生首页二次收敛为“作业列表 + 右上角登录/身份”：公共题库作为第一份公开作业，老师作业在登录班级后出现。
- 作业入口改为 PTA 式做题工作台：左侧题目列表，中间题目内容，右侧代码编辑；提交反馈用弹窗展示测试点、错误提示、优化提示和 Coach 追问。
- 旧邀请码入口保留兼容，但不再作为学生端主路径和主导航文案。
- 为学生端补齐薄接口：学生登录、按学生画像获取可见作业。接口只服务身份与作业可见性，不改变数据库结构、评测链路或 AI 诊断逻辑。

## Capabilities

### New Capabilities

- `student-teacher-learning-flow`: 约束学生端、教师端、公共题库、班级登录、作业详情和题目页之间的路由与可见性链路。

### Modified Capabilities

- 无。

## Impact

- OpenSpec：新增本变更的 proposal、design、tasks 和规格。
- 后端：在现有课堂服务上新增学生登录与学生可见作业接口；复用现有 `StudentProfile`、`ClassGroup`、`Assignment`、`AssignmentTask` 和身份 key 逻辑。
- 前端：调整 `App.tsx` 路由和导航；将 `/app` 兼容跳转到学生端；重构学生端首页、登录、公开作业入口、PTA 式做题工作台和结果弹窗；更新 API client、session storage 和 browser smoke。
- 样式：新增少量学生端路由布局样式，复用现有按钮、面板、状态标签和列表组件风格。
- 非影响范围：不改判题、提交、AI 诊断、教师分析、数据库迁移和外部模型评测链路。
