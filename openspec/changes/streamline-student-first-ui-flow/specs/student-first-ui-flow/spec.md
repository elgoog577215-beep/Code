# student-first-ui-flow 规格增量

## ADDED Requirements

### Requirement: 学生端必须以下一步行动为首屏主目标

`/app/student` 在已有作业和已确认身份时，MUST 优先展示身份摘要、当前作业、下一题和进入题目的主操作。

#### Scenario: 学生已确认身份

- **WHEN** 学生打开 `/app/student`
- **THEN** 页面首屏显示学生身份摘要、作业标题、完成进度和下一题主操作
- **AND** 题目列表显示题号、题名、难度、状态和开始/继续/复盘操作
- **AND** 学习记录、每题详细状态和长期信号不与下一题主操作竞争

### Requirement: 学生端详细学习记录必须作为第二层信息

提交次数、每题状态、长期学习信号和追问摘要 MUST 默认放入折叠或次级区域。

#### Scenario: 学生已有学习轨迹

- **WHEN** 学生进入作业页
- **THEN** 完成度和提交次数可以在折叠摘要中查看
- **AND** 页面主视觉仍然聚焦下一题和题目列表

### Requirement: 题目页提交后必须只突出一个主要处理项

`/app/problem/:id` 提交后 MUST 把“本次先做”的处理项作为反馈区最明显内容，下一问紧贴该处理项。

#### Scenario: 提交未通过

- **WHEN** 学生提交代码且评测未通过
- **THEN** 反馈区突出一个主处理项
- **AND** 如果有公开失败证据，应紧贴主处理项展示
- **AND** 下一问显示在主处理项附近
- **AND** 测试点详情、诊断详情、作业记录和尝试记录默认折叠

### Requirement: 导航必须按使用上下文降噪

顶部导航 MUST 根据当前页面上下文优先展示相关入口，避免学生主流程中出现同等权重的教师入口。

#### Scenario: 学生或题目页面

- **WHEN** 用户位于 `/app/student` 或 `/app/problem/:id`
- **THEN** 顶部导航优先展示当前作业、题库和邀请码
- **AND** 教师入口不作为同等权重主导航项出现

#### Scenario: 教师页面

- **WHEN** 用户位于教师相关页面
- **THEN** 顶部导航优先展示课堂、管理和题库
- **AND** 教师首页首屏仍展示课堂 KPI、需关注学生和高频问题

### Requirement: 响应式页面不得牺牲主操作可用性

关键学生页面、题目页和教师页 MUST 在 390px、820px、1440px 下无横向溢出，主要按钮不低于 44px。

#### Scenario: 移动端学生主流程

- **WHEN** 浏览器宽度为 390px
- **THEN** 学生下一题主操作、题目列表操作和题目页提交按钮可见且不被横向裁切
