# 学生优先 UI 主流程再简化

## Summary

本轮目标是把当前前端从“功能已经分区”进一步收敛到“同学最顺手”：每个页面只突出一个主动作，其余信息进入列表、折叠详情或教师专属页面。变更只涉及前端信息层级、导航分布、样式和浏览器 smoke；不新增、不删除、不重命名后端 API，不改变 AI 诊断、评测链路或数据库。

## Motivation

当前页面已经比早期安静，但学生端仍把身份、当前作业、题目列表、提交状态和学习记录放在同一视觉层级；题目页提交后也容易让反馈、测试点、历史记录和下一问互相竞争。学生真正需要的是三秒内知道：我是谁、当前作业是什么、下一步点哪里、提交后先改什么。

## Scope

- 学生端：改为“身份条 / 下一题 / 题目列表 / 学习记录折叠”的顺序。
- 题目页：保持“题目 / 代码 / 提交反馈”三段式，但提交反馈只突出一个主反馈，下一问紧贴主反馈。
- 导航：根据当前路径显示更贴近上下文的入口，学生/题目页弱化教师入口。
- 教师端：首屏保留课堂 KPI、需关注学生和高频问题；AI 质量继续作为摘要与折叠信息。
- 验证：同步 browser smoke，覆盖新增上下文导航与学生优先主流程。

## Non-Goals

- 不做登录权限系统。
- 不改后端 API、数据库、AI 诊断逻辑、评测链路或字段语义。
- 不引入新 UI 框架或新图标库。
- 不重写题库、教师管理、题目编辑等低频页面。
