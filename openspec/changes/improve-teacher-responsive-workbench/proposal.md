## Why

当前教师端已经完成课堂流程简化，但响应式布局仍偏向单列收束：教师管理页被限制在较窄容器内，任务卡始终纵向堆叠；作业中心在桌面宽屏下没有充分利用横向比较空间；作业详情在中大屏过早退化为纵向列表。这样在教师使用大屏、笔记本外接显示器或教室电脑时，会增加无效滚动，也让“管理、观察、校正”这些工作台任务显得抽象。

本变更需要把教师端从“手机优先的竖向页面”补齐为“尺寸自适应的教学工作台”：大屏横向展开，中屏保留关键两列，小屏回到单列流程。

## What Changes

- 统一教师端响应式断点策略：390px 手机、820px 中屏、1440px 桌面必须都有明确布局目标。
- 教师管理页在桌面三列展示创建班级、导入名单、导入题目；中屏两列并让题目导入跨行；手机保持纵向流程。
- 作业中心保留行式作业列表，但扩大桌面容器，让作业名称、课堂指标和进入动作横向比较。
- 作业详情在桌面形成工作台布局，让整体统计、学生情况和题目表现减少纵向堆叠；中屏保留学生列表和学生报告的双列可读性；手机单列。
- 更新浏览器 smoke，验证三类教师页面在 390px、820px、1440px 下无横向溢出，并覆盖桌面横向布局断言。
- 不改变后端 API、共享类型、路由或已简化的教师管理功能范围。

## Capabilities

### New Capabilities

- `teacher-responsive-workbench`: 定义教师端在手机、中屏和桌面宽屏下的工作台布局、无横向溢出和触控尺寸要求。

### Modified Capabilities

无。

## Impact

- Frontend: `TeacherPage`、`AssignmentDetailPage`、`TeacherManagementPage` 的 CSS 布局规则，以及 `browser-smoke` 响应式断言。
- OpenSpec: 新增教师端响应式工作台能力说明、设计和任务。
- APIs / Backend / Database: 无变更。
- Validation: 运行 `npm run typecheck` 和 `npm run smoke:browser`。
