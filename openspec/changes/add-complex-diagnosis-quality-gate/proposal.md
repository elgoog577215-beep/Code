## Why

上一轮已经补齐了 100 条复杂学生提交样本，并让前 24 条进入 live eval；但复杂样本里的 6 个关键质量指标目前还停留在 fixture 元数据，报告只统计标签命中和证据非空。这样会低估“多错因里选最该先教根因”的真实难度，也难以判断外接模型调优是否真正提升教育 agent 能力。

## What Changes

- 新增复杂诊断质量评分器，把 `primaryRootCauseHit`、`teachingPriorityCorrect`、`secondaryIssuesNotOverweighted`、`distractingSignalsIgnored`、`evidenceGrounded`、`noFullSolutionLeak` 变成可计算结果。
- 将复杂质量结果写入 live model eval entry 和 summary，使报告能展示复杂样本通过数、指标通过数和平均分。
- 让复杂样本的 quality baseline draft 和 baseline regression gate 能保留并检查复杂质量信号。
- 增加离线测试，验证好输出能全量通过，偏离主错因、过度强调次错因、引用缺失和泄露答案会被门禁捕获。

## Capabilities

### New Capabilities

- `complex-diagnosis-quality-gate`: 复杂诊断样本 SHALL 被逐项评分，并作为 live eval 与 baseline regression 的质量门禁。

### Modified Capabilities

- `autogenerated-complex-eval-fixtures`: 复杂样本已有的 expectedMetrics SHALL 不只是元数据，还 SHALL 驱动可计算评测结果。

## Impact

- 影响测试/评测侧代码：复杂 fixture loader、live model eval report、quality baseline draft、baseline regression gate、相关 JUnit。
- 不修改生产诊断链路，不改变外接模型调用方式，不引入新依赖。
- 生成的 report JSON 会新增复杂质量字段；旧字段保持兼容。
