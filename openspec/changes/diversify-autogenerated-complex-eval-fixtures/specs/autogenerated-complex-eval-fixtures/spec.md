## MODIFIED Requirements

### Requirement: 自动生成复杂诊断样本

系统 SHALL provide an autogenerated complex student submission fixture set for diagnosis evaluation without relying on crawled student code or manual review, and the set SHALL satisfy semantic diversity gates.

#### Scenario: complex fixture set is generated

- **GIVEN** the deterministic complex fixture generator
- **WHEN** the generated fixture resource is loaded
- **THEN** it SHALL contain exactly 100 cases
- **AND** each case SHALL include a stable `caseId`
- **AND** each case SHALL include a `generatorSpecId`
- **AND** each case SHALL include 40-80 lines of Python source code
- **AND** the set SHALL include at least 14 generator template slugs
- **AND** the set SHALL include at least 12 primary fine-grained diagnosis tags

#### Scenario: generated case carries high-quality labels

- **WHEN** a generated complex case is inspected
- **THEN** it SHALL include `primaryRootCause`
- **AND** it SHALL include at least one secondary issue
- **AND** it SHALL include distracting signals
- **AND** it SHALL include `expectedTeachingPriority`
- **AND** it SHALL include required evidence refs

### Requirement: 自动执行验证样本证据

系统 SHALL verify generated complex fixtures with local Python execution evidence.

#### Scenario: correct solution is verified

- **WHEN** a generator seed builds a correct solution
- **THEN** the correct solution SHALL pass all seed tests

#### Scenario: buggy submission is verified

- **WHEN** a bug recipe builds a generated student submission
- **THEN** the buggy submission SHALL produce the expected failure verdict
- **AND** the first failed case SHALL be preserved in `caseResults`
- **AND** the case SHALL include evidence refs that identify the first failure and injected root cause

### Requirement: 复杂样本服务根因优先级评测

系统 SHALL make complex fixtures evaluate whether the model selects the most teachable root cause among multiple signals.

#### Scenario: model evaluates a complex case

- **WHEN** a complex case is included in model diagnosis eval
- **THEN** the eval case SHALL expose expected issue tags and fine-grained tags
- **AND** the expected fine-grained tags SHALL include the primary root cause first
- **AND** quality metadata SHALL include `primaryRootCauseHit`, `teachingPriorityCorrect`, `secondaryIssuesNotOverweighted`, `distractingSignalsIgnored`, `evidenceGrounded`, and `noFullSolutionLeak`
