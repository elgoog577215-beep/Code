## Why

当前前端已经具备学生作业、题目练习、教师过程看板、AI 质量和教师校正等能力，但这些能力在首屏同时出现时会增加课堂使用的判断成本。现在需要把界面从“功能都展示出来”收束为“下一步动作最清楚”，让学生和教师在课堂场景中更快完成关键判断。

## What Changes

- 简化学生端作业页，保留邀请码、身份、当前作业、题目列表和提交状态，弱化低频身份详情。
- 简化题目页为题目与样例、代码编辑、提交反馈三段式，突出一个主要处理项，保持作业记录和尝试记录为二级信息。
- 简化教师工作台首屏层级，优先展示作业选择、课堂 KPI、需关注学生和高频问题；AI 质量能力保留为摘要与折叠详情。
- 简化教师管理、题目编辑和班级概览的低频设置，把导入、保存、讲评等主动作放在更清楚的位置。
- 收敛全局视觉噪音，减少徽章、边框和卡片堆叠，保持深色和浅色主题可读。
- 不改变后端 API、数据库、AI 诊断、评测或推荐闭环语义。

## Capabilities

### New Capabilities
- `campus-teaching-ui-simplification`: 定义校园教学界面简化后的页面主次、二级信息折叠、移动端触控与验证要求。

### Modified Capabilities

无。

## Impact

- Frontend: React 页面与 `styles.css`，覆盖学生端、题目页、教师工作台、教师管理、题目编辑和班级概览。
- OpenSpec: 新增校园教学 UI 简化能力说明、设计和任务。
- APIs / Backend / Database: 无变更。
- Validation: 运行前端 typecheck、build、browser smoke，并人工复查移动端和深色模式。
