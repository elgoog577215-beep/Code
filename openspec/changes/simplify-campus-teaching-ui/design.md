## Context

本项目是校内在线判题与 AI 教育 agent 工具。界面应服务课堂现场，而不是展示 AI 产品能力。已有学生端和教师端已经能完成主要流程，但随着 AI 质量、推荐闭环、教师校正和 eval 候选能力增加，页面首屏出现了更多状态卡片和徽章，降低了“下一步动作”的清晰度。

## Goals

- 学生端优先回答：我是谁、做哪题、现在状态是什么、下一步做什么。
- 教师端优先回答：当前作业怎样、谁需要关注、全班共性问题是什么、AI 是否需要处理。
- 保留 AI 质量、教师校正、eval 候选等能力，但默认退到摘要或折叠区。
- 保持现有路由、接口和类型兼容。

## Design

采用“课堂工具型简化”而不是极简删除。每个页面保留一个主任务区，二级信息使用已有 `details` 折叠模式承载。视觉语言继续沿用克制的工具台风格，减少装饰和状态标签密度。

### 学生端

学生作业页保持两种状态：
- 未确认身份：身份确认在前，题目列表可见但行动按钮明确受限。
- 已确认身份：题目列表为主角，身份信息降级为轻量身份条，提交状态作为辅助。

题目卡片只保留题号、标题、难度/最新状态和开始按钮，不展示额外解释。

### 题目页

题目页保持三段式：
- 题目与公开样例。
- 代码编辑和提交。
- 提交反馈。

提交反馈区先展示处理项，再展示测试点摘要、首个失败点、AI 反馈和下一问。作业记录与尝试记录保持折叠，避免抢占首屏。

### 教师端

教师工作台首屏顺序调整为：
1. 作业选择和课堂 KPI。
2. 需要关注学生。
3. 高频问题和讲评参考。
4. AI 质量摘要。

AI 质量只默认展示状态、样本量、优先改进和评测沉淀摘要。质量维度、eval 候选和 fixture 草稿进入折叠详情。教师校正仍由学生行的“校正”按钮触发。

### 管理与编辑

教师管理继续保留名单、题目、系统三类任务，但每类只突出当前主操作。题目编辑页把基础题目信息、题面和测试点作为主流程，教学增强字段保持折叠。

## Risks

- 现有 `browser-smoke` 依赖多个 CSS selector，简化时必须保持关键 class 名兼容。
- 当前工作区已有 AI 质量和教师介入相关未提交改动，UI 简化必须顺着这些字段展示，不能回退。
- `styles.css` 已经较大，本轮只做必要收敛，避免大规模重写造成回归。

## Validation

- `npm run typecheck`
- `npm run build`
- `npm run smoke:browser`
- 检查桌面、移动和深色截图：无横向溢出、主动作清楚、AI 质量不压过课堂过程。
