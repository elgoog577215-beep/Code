## 执行原则

本清单约束产品目标、事实边界和验收结果，不预先锁死类名、文件数量或内部实现。每个阶段开始前必须重新读取当前代码并优先复用已有能力；若真实代码与设计假设冲突，先更新 `design.md`、增量规格和本清单，再继续实现。允许调整同阶段任务顺序，但不得跳过阶段门禁和最终验收。

## 1. 现状复核与成长契约

- [x] 1.1 复核现有提交、测试点、诊断事实、生命周期、学生反馈、历史接口和教师下钻代码，记录可直接复用能力与实际缺口；发现规格假设不成立时先修订 OpenSpec
- [x] 1.2 定义学生与教师共享的成长状态和结构化证据契约，覆盖有效性、可比性、比较提交、评测差值、问题变化、优先问题、规则版本和数据完整性
- [x] 1.3 建立代表性验收样本矩阵：首次、重复、分析失败、明显进步、混合进步、问题转移、停滞、回退、最终通过，以及“10 个问题降到 5 个且其中 1 个新增”

阶段门禁：契约只描述稳定事实和可验证结果；不得引入成长分数、代码行数判断、第二套前端分类或新的诊断 Agent。

## 2. 有效比较链与成长投影

- [x] 2.1 实现同一学生、题目和作业范围内的全量提交链与最近可比较有效基线，隔离不同作业和公共题库，并保留重复、身份缺失和分析不完整记录
- [x] 2.2 基于判题、可比测试点和问题生命周期实现确定性成长分类与并列证据，严格区分持续、新增、未观察、恢复、复发和不可比较
- [x] 2.3 将错误/阻塞问题与可选提升建议分开计量，使成长事实独立于外部模型建议状态，并支持生命周期回填或教师校正后重算
- [x] 2.4 使用样本矩阵完成后端单元测试，验证中间重复或分析失败提交不会打断有效比较链，且低置信度身份不会被表述为正式同一问题

阶段门禁：后端能够仅凭保存事实稳定回答“当前提交相对哪次有效提交发生了什么变化”，模型失败不得改变已有确定性结论。

## 3. 学生与教师共享读取接口

- [x] 3.1 批量组装提交、测试点统计、生命周期转换和完整性状态，避免按历史节点逐条读取反馈形成 N+1 查询
- [ ] 3.2 兼容扩展学生反馈、本题历史和全局提交详情响应，返回成长摘要、实际比较提交和全量可回看节点；长历史支持分页或增量加载
- [x] 3.3 兼容扩展教师题目学生聚合与学生单题详情，复用同一成长投影，并让教师校正明确绑定被查看的提交与反馈版本
- [ ] 3.4 完成接口集成测试，覆盖学生权限、作业范围隔离、公共题库、历史反馈版本、教师校正重算、旧数据兼容和批量查询性能

阶段门禁：同一提交在学生端和教师端返回一致的成长状态、比较基线与问题计数；两端不得各自推断。

## 4. 学生当前结果页

- [x] 4.1 接入共享成长类型和双语状态映射，在判题结果下实现视觉稿中的连续数据框、评测趋势、知识点排名、问题变化条与成长摘要
- [x] 4.2 按持续、复发、新增组织当前完整问题，将未观察与恢复放入可展开历史变化区，并把可选提升建议独立展示
- [x] 4.3 覆盖首次、重复、不可比较、回退、通过、AI 生成中和 AI 失败状态；取消成长区域对 `MODEL/READY` 的硬依赖，保留评测、代码和证据跳转
- [x] 4.4 完成结果页关键状态测试和人工检查，确保学生第一眼能回答“这次怎么样、变化在哪里、下一步改什么”

阶段门禁：模型建议失败时仍能展示可靠成长事实；页面不能只给鼓励而隐藏新增、复发或回退。

## 5. 学生单题成长时间线

- [x] 5.1 将题目页最近提交列表升级为最新在上的全量成长时间线，突出有效节点并降低重复、不可比较节点的视觉权重，但不删除真实记录
- [x] 5.2 打通任意时间线节点到该次评测、代码、成长摘要和保存反馈版本的回看，并在全局提交详情提供对应成长信息和本题轨迹入口
- [ ] 5.3 验证长历史加载、键盘操作、屏幕阅读顺序和移动端单列展示，确保状态不只依赖颜色表达

阶段门禁：学生能够从当前结果回到任意一次提交，并明确看到该次实际比较的是哪条有效提交。

## 6. 教师单题学生轨迹

- [ ] 6.1 在教师题目学生列表展示原始提交、有效修改、最新成长状态和持续/新增/复发关注信号，保持“作业 → 题目 → 学生”的既有下钻路径
- [ ] 6.2 在教师学生单题详情加入同源逐提交时间线，保留逐问题轨迹、评测证据和 AI 校正，并支持从任意节点打开前后代码与诊断证据
- [ ] 6.3 完成教师视图测试，确保改善、停滞、问题转移、回退和复发使用客观观察性文案，不宣称 AI 导致变化或替教师作决策

阶段门禁：教师能够判断学生是否持续卡住、是否出现回退或复发、是否需要介入，并能追溯具体提交证据。

## 7. 整体验收与收束

- [x] 7.1 运行后端测试、前端关键状态测试、TypeScript、lint 和构建，修复所有由本变更引入的失败
- [ ] 7.2 使用样本矩阵完成学生当前结果、学生时间线、全局提交详情和教师学生轨迹的真实浏览器验收
- [x] 7.3 检查中文与英文、明暗主题、桌面/平板/手机下无中文残留、文字遮挡、横向溢出或仅颜色可辨状态
- [ ] 7.4 对照 proposal、design 和全部增量规格复核实际实现，更新偏差与项目记忆，运行 `openspec validate --all` 后再归档变更

阶段门禁：不是“任务全部打勾”即完成；只有学生和教师能够基于同一份真实证据读懂单题成长，且自动化与浏览器验证均通过，才允许收束。
