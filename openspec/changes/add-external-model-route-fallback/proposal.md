# 新增外部模型备用路由

## 背景

当前外部模型 live eval 已经能区分模型质量问题和运行容量问题。最近一次 10 条真实长代码评测显示，已完成输出的诊断信号、证据、安全和教学动作均达标，但整体完成率被主路由额度或限流拖低。

如果生产环境只有单一外部模型路由，学校真实使用时会把额度、限流、模型暂不可用误判为教育 agent 不会分析。下一步应把主路由失败后的备用路由纳入原有诊断、追问、成长报告链路。

## 目标

- 让提交诊断、Coach 追问、成长报告共用外部模型备用路由能力。
- 主路由因 quota、限流、模型不支持、网络或空响应失败时，可以尝试备用 OpenAI-compatible 路由。
- 预算门禁仍按 provider/model 隔离，主路由 guard 打开时不阻断备用路由。
- live eval 的备用路由配置与服务真实调用配置保持一致。

## 非目标

- 不重写提示词、标准库或诊断协议。
- 不引入多模型投票、自动择优、结果融合。
- 不把备用路由失败计入模型质量问题。

## 影响范围

- 外部模型调用链路。
- 提交诊断 AI invocation 元数据。
- Coach agent 外部模型调用。
- live eval 服务构造与报告解释。
