# 新增 AI 路由健康预检

## 背景

当前 10 条长代码 live eval 已证明：已完成的外部模型输出质量达标，但单一路由 quota 会让整体完成率掉到 20%。系统已经支持 `ai.routes` 多路由池，但如果部署者没有配置备用 key 或 route pool，问题仍会在学生提交时才暴露。

因此需要一个只读健康预检接口，部署前就能看到当前 AI 路由是否单点、是否缺 key、是否只有本地兜底风险。

## 目标

- 暴露系统级 AI 路由健康画像。
- 不调用外部模型，不消耗额度。
- 不返回任何 API key 或可逆密钥信息。
- 明确提示当前风险：未启用、无可用路由、单一路由、多路由可用。

## 非目标

- 不做真实模型连通性探测。
- 不存储路由健康历史。
- 不替代 live eval 的真实质量评测。

## 影响范围

- system 模块只读接口。
- 外部模型路由配置解析。
- 部署与教师排障时的可观测性。
