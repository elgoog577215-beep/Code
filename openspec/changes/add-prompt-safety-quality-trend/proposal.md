## 背景

项目已经把提示安全事件接入了单作业 AI 质量概览，也能把高泄题诊断和提示安全降级导出为安全 eval 草稿。当前缺口是：跨作业 AI 质量趋势仍只展示低置信度、高泄题、教师校正和课堂介入成效，老师看不到提示安全事件是否集中在某些作业、某个模型来源或某个提示版本上。

这会让提示安全闭环停在“单次发现”和“人工导出”之间，缺少持续观测能力。对于教育 agent，安全不是一次性门禁，而是长期质量指标。

## 变更目标

- 在跨作业 AI 质量趋势中新增提示安全事件计数、提示安全降级计数、高风险降级计数和提示安全事件率。
- 在作业趋势点中显示每个作业的提示安全事件数量和安全降级数量。
- 在模型/提示来源分段中归因高泄题风险和提示安全降级，帮助判断问题是否集中于某个 source/version。
- 更新教师端 API 类型，并在跨作业趋势区展示提示安全事件与安全降级读数。
- 用后端测试验证跨作业聚合、LOW 安全检查过滤和来源分段归因。

## 非目标

- 不新增新的安全判定策略；本变更只消费已有 `answerLeakRisk` 和 `HintSafetyCheck`。
- 不改变 `HintSafetyService` 的安全降级逻辑。
- 不重做教师端趋势 UI 布局；前端本轮只补类型，避免和已有 UI 简化变更互相踩踏。

## 影响范围

- 后端：`AiQualityTrendService`、`AiQualityTrendResponse`、相关测试。
- 持久层：复用 `HintSafetyCheckRepository.findBySubmissionIdIn`。
- 前端：`AiQualityTrend`、`AiQualityTrendPoint`、`AiQualitySourceSegment` 类型新增可选字段；跨作业趋势卡片展示提示安全读数和作业/来源安全标记。
